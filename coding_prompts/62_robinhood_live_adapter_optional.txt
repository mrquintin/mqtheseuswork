Goal: optional Robinhood adapter for live equity trading. The
founder named Robinhood explicitly. Alpaca (prompt 60) is the
recommended live broker; Robinhood is offered as an alternative
behind explicit flags, with the caveats documented prominently.

Reality check the operator must read before flipping the flag:
 - Robinhood does NOT publish a supported public retail trading
   API. Any integration depends on unofficial reverse-engineered
   libraries (e.g. `robin_stocks`) which Robinhood periodically
   breaks with app releases.
 - Robinhood may detect, throttle, or close accounts that
   automate trading.
 - The eight-gate safety contract still applies; this is not a
   bypass.

For these reasons the Robinhood adapter is OFF by default, behind
its own enable flag separate from the global live-trading flag,
and the operator-console UI shows a yellow banner naming the
risks whenever it is enabled.

C. Live equity engine
   `noosphere/noosphere/equities/live_trader.py`:
   - `async submit_live_equity_order(store, position_id, *,
                                      operator_id,

     1. Load `EquityPosition` + linked `EquitySignal` +

     2. Build `GateContext` from env + portfolio state.
