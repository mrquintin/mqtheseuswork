## The problem

The Noosphere peer review system generates rich structured findings — each reviewer produces findings with severity levels (info, minor, major, blocker), categories, detailed explanations, evidence, and suggested actions. But the Codex UI only shows the reviewer's overall verdict and a free-text commentary. The structured findings, which are the most actionable part of a peer review, are invisible.

## Your task

Surface the structured findings from peer reviews in the Codex UI. This may require:
- Adding a `findings_json` column to the `peer_review` table (or checking if it already exists)
- Updating the Noosphere CLI output to include findings
- Updating the Codex fetch function and UI to display them

## Step-by-step

1. **Check the database schema.** Look at what columns the `peer_review` table actually has. It's queried via raw SQL in `fetchPeerReviews`:
   ```sql
   SELECT id, conclusion_id, reviewer_name, verdict, commentary, created_at FROM peer_review
   ```
   Check if additional columns exist (like `findings_json`, `severity`, etc.) that are simply not being selected.

2. **If findings are stored in the DB**, update the SQL query to include them:
   ```sql
   SELECT id, conclusion_id, reviewer_name, verdict, commentary, findings_json, created_at FROM peer_review
   ```
   Update the `PeerReviewRecord` type to include:
   ```typescript
   findings: Finding[];
   ```
   Where `Finding` is:
   ```typescript
   interface Finding {
     severity: "info" | "minor" | "major" | "blocker";
     category: string;
     detail: string;
     evidence: string[];
     suggestedAction: string;
   }
   ```

3. **If findings are NOT in the DB**, you have two options:
   - **Option A (preferred):** Add `findings_json` to the raw SQL table. Add it to the Noosphere CLI output so future reviews include it. Existing reviews will have null findings.
   - **Option B:** Keep it as-is and note in the UI that structured findings are available in Noosphere but not yet synced to the Codex.

4. **Update the peer review tab** to render findings when available:
   ```tsx
   {r.findings && r.findings.length > 0 && (
     <details style={{ marginTop: "0.5rem" }}>
       <summary style={{ cursor: "pointer", fontSize: "0.7rem", color: "var(--parchment-dim)" }}>
         {r.findings.length} finding{r.findings.length > 1 ? "s" : ""}

       </summary>
       <div style={{ marginTop: "0.35rem" }}>
         {r.findings.map((f, i) => (
           <div key={i} style={{
             padding: "0.4rem 0.75rem",
             marginBottom: "0.25rem",
             borderLeft: `2px solid ${severityColor(f.severity)}`,
             fontSize: "0.75rem",
           }}>
             <div style={{ display: "flex", gap: "0.5rem", alignItems: "center" }}>
               <span style={{ color: severityColor(f.severity), textTransform: "uppercase", fontSize: "0.6rem" }}>
                 {f.severity}
               </span>
               <span style={{ color: "var(--parchment-dim)", fontSize: "0.6rem" }}>

             )}
           </div>
         ))}
       </div>
     </details>
   )}
   ```

5. **Add a severity color helper** (or import from `@/lib/colors` if prompt 18 has been applied):
   ```typescript
   function severityColor(s: string): string {
     switch (s) {
       case "blocker": return "var(--ember)";
       case "major": return "var(--amber)";
       case "minor": return "var(--parchment)";
       default: return "var(--parchment-dim)";
     }
   }
   ```
