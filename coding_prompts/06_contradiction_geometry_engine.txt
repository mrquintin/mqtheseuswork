Implement:

A. `noosphere/noosphere/coherence/contradiction_engine.py`
   (the new canonical engine; the old multi-heuristic module is
   demoted to a legacy compat shim):
   - `class ContradictionEngine`:
     - `async detect(principle_a, principle_b, *, store) ->
                                          ContradictionResult`:
       1. Resolve embeddings for both principles using the
          embedding family the benchmark selected.
       2. Apply the contradiction-geometry transform (the
          Householder-style projection if the ablation showed it
          best; otherwise the alternative the benchmark winner
          uses). The decision is locked HERE — there is one
          canonical method, not six.
       3. Compute the contradiction score: a scalar in [0, 1]
          where 0 = independent / compatible, 1 = strict
          contradiction.
       4. Compute a confidence band (low, high) for the score.
          Bands come from the benchmark's calibration data.
       5. If score > `CONTRADICTION_THRESHOLD` (default 0.65),
          generate a structured contradiction reason: which
          axis-of-disagreement the geometry highlights, with a
          short human-readable description ("the principles
          disagree on whether X causes Y").
          The reason is produced by a brief Haiku call that
          takes both principle texts + the geometric signal and
          MUST GROUND its explanation in actual disagreement
          between the texts (not in the geometry itself —
          geometry detects, language explains).
       6. Return ContradictionResult:
          { score: float,
            confidence_low: float,
            confidence_high: float,
            verdict: "CONTRADICTORY" | "INDEPENDENT" | "COHERENT",
            axis: str | null,
            human_explanation: str | null,
            detection_method: str,  # version-stamped
            detected_at: datetime }

   - `async batch_detect(principle_pairs, *, store, max_concurrency=4)
                                       -> list[ContradictionResult]`

B. Versioned detection method. The contradiction engine carries
   `detection_method` as a versioned string (e.g.
   `"geometry/householder/v2"`). This appears on every persisted
   ContradictionResult so future analyses can group by method
   version. New rolls of the benchmark can produce a new version
   without invalidating prior results.

C. Persistence. The existing `StoredContradiction` (or analog)
   table is extended with:
   - `score` float
   - `confidence_low`, `confidence_high` floats
   - `axis` text nullable
   - `human_explanation` text nullable
   - `detection_method` text
   Old rows retain whatever heuristic produced them; new rows
   carry the new fields. The Prisma migration is additive.

F. Operator surface. The `/(authed)/contradictions` page (extend

G. CLI:
   - `noosphere contradiction detect --a <id> --b <id>` — single
     pair.
   - `noosphere contradiction sweep --since <iso>` — re-run the
     engine over all principles added since a timestamp.
   - `noosphere contradiction methods` — list available method
     versions and their benchmark stats.

H. Tests:
   - Engine produces stable scores on a fixture pair across
     repeated runs (determinism within tolerance).
   - Contradictory pair fires above threshold with sensible
     axis text.
   - Independent pair scores low (< 0.3).
   - Coherent / supporting pair also scores low; engine
     distinguishes COHERENT from INDEPENDENT via the verdict
     enum.
   - Adversarial fixture: principles whose texts are
     semantically identical but lexically different → engine
     does NOT flag as contradictory.
   - Method-version stamping is correct on persisted rows.
   - DISPUTE action records the dispute and surfaces it on the
     methodology page.

Constraints
- ONE detection method ships as canonical. The era of "we have
  six heuristics that vote" is over.
- The contradiction engine NEVER auto-resolves a contradiction.
  Resolution is source-driven (prompt 08).
- The Haiku call that produces the human explanation MUST cite
  the verbatim disagreement; if it cannot, it produces a null
  explanation and the contradiction is flagged with axis but
  no narrative.
- Calibration: the engine's confidence bands must match its
  actual reliability ± 0.10 — verified by a benchmark integration
  test against the QH ground-truth fixtures.
