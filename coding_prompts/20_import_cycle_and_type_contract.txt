Goal: catch the two highest-impact silent failure modes Round 19

What "import cycle" means in this codebase:

- `noosphere.algorithms.*` may import from `noosphere.core`,
  `noosphere.coherence`, `noosphere.store`. It MUST NOT import
  from `noosphere.synthesizer` or `noosphere.portfolio_agent`.
- `noosphere.synthesizer.*` may import from `noosphere.algorithms`,
  `noosphere.coherence`, `noosphere.core`. It MUST NOT import
  from `noosphere.portfolio_agent`.
- `noosphere.portfolio_agent.*` may import from `noosphere.synthesizer`,
  `noosphere.bets`, `noosphere.forecasts`, `noosphere.equities`.
  It is a CONSUMER of upstream modules; it must not be imported
  by any of them.
- `noosphere.bets.*` is a pure schema layer plus per-kind
  resolvers. It depends on `noosphere.forecasts` and
  `noosphere.equities` for MARKET resolution; nothing depends
  on bets except portfolio_agent.
- `noosphere.knowledge_graph.*` reads from everything but writes
  to nothing except its own snapshot tables.

What "type contract" means:

- Every FastAPI route in `current_events_api` that returns JSON
  must have a Pydantic response model.
- Every Next.js API route in `theseus-codex/src/app/api/` that
  consumes a FastAPI route must have a TypeScript type that
  matches.
- The match is enforced by code-generation: a Python script
  walks the FastAPI app and emits TypeScript types into
  `theseus-codex/src/lib/_generated/api/`. The generated file
  is committed; CI fails if regeneration produces a diff.
