# Codex Prompt 4 of 18 ŌĆö Kalshi market ingestor (read-only public market list)

**Sequence:** prompt 4 of 18 (Round 10, Wave A). Depends on 03. Report back with: a sample serialized `ForecastMarket` row, the categories your ingestor accepts, and how the test fixture handles Kalshi's RSA-signed auth requirement.

**Why this prompt exists.** Kalshi's *market metadata* endpoints (`/exchange/announcements`, `/markets`, `/series`) are technically public, but their API requires authenticated requests for everything beyond the announcements feed. We treat ingestion as auth-required-but-read-only: a service-account API key is needed, signed via RSA. The key is read from env; if absent, the ingestor is a no-op and logs a single line. Live trading happens later (prompt 09); this prompt only reads.

1. Read `noosphere/noosphere/forecasts/_polymarket_client.py` from prompt 03 ŌĆö your Kalshi client mirrors that shape (httpx async, retry, no global state) but adds RSA request signing.
2. Read `noosphere/noosphere/forecasts/polymarket_ingestor.py` ŌĆö copy the `IngestResult` dataclass, the dedupe rules, and the change-detection threshold. The two ingestors expose the same interface; only the source-specific parsing differs.
3. Read Kalshi API docs for the request-signing scheme: a `KALSHI-ACCESS-TIMESTAMP`, `KALSHI-ACCESS-KEY` (key id), and `KALSHI-ACCESS-SIGNATURE` (RSA-PSS-SHA256 over `timestamp + method + path`). Confirm the algorithm ŌĆö if their docs disagree with this prompt, FOLLOW THE DOCS and note the deviation.

### `noosphere/noosphere/forecasts/_kalshi_client.py`

```python
class KalshiClient:
    def __init__(self, *, base: str, key_id: str, private_key_pem: str,
                  timeout_s: float = 15.0): ...

    async def list_markets(self, *, status: str = "open",
                            limit: int = 100, cursor: str | None = None,
                            ) -> tuple[list[dict], str | None]:
        """Returns (markets, next_cursor)."""

    async def get_market(self, ticker: str) -> dict | None: ...
    async def get_event(self, event_ticker: str) -> dict | None: ...
```

Implementation notes:
- RSA private key parsing via `cryptography.hazmat.primitives.serialization.load_pem_private_key`.
- Sign as: `signature_b64 = base64( RSA-PSS-SHA256(  f"{timestamp}{method.upper()}{path}".encode() ) )`. If their docs require a different format, follow the docs.
- Compute timestamp as ms since epoch (per their spec).
- Headers: `KALSHI-ACCESS-KEY`, `KALSHI-ACCESS-SIGNATURE`, `KALSHI-ACCESS-TIMESTAMP`, `Accept: application/json`.
- Retry policy identical to Polymarket client.
