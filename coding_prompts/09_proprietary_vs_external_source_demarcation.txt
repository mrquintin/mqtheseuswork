Implement:

A. New entity dimension: `ProvenanceKind` enum on every
   ingested artifact and every derived principle/conclusion:
   - `PROPRIETARY` — the firm authored it. Default.
   - `ENDORSED_EXTERNAL` — external, but explicitly endorsed
     by the firm as representative of its thinking. Founder must
     mark this at upload time.
   - `STUDIED_EXTERNAL` — external, used as a reference/test
     case, NOT endorsed (e.g., a piece the firm critiques).
   - `OPPOSING_EXTERNAL` — external, explicitly disagreed-with
     but kept for contradiction-testing value (e.g., a piece the
     firm thinks is wrong but wants to be able to argue against).
   - Provenance is set at UPLOAD TIME and is NOT inferred.

B. Upload flow change:
   - The upload UI presents a required radio:
     `Proprietary | Endorsed external | Studied external |
      Opposing external`.
   - For external choices, a required text field "Why is this
     in our library?" — short rationale (≥ 30 chars), persisted
     on the artifact row.
   - The agent NEVER infers provenance from content. The founder
     decides.

C. Storage. Artifacts, claims, principles, conclusions, and
   algorithms ALL inherit `provenance` from their root source
   artifact. New columns added wherever applicable. Indexes on
   `(organizationId, provenance, createdAt DESC)` to make
   filtered queries fast.

D. Oracle / synthesis API extension. Every Oracle / synthesis
   call accepts a `provenance_filter` parameter:
   ```
   { include_proprietary: bool (default true),
     include_endorsed_external: bool (default true),
     include_studied_external: bool (default false),
     include_opposing_external: bool (default false),
     proprietary_weight: float (default 2.0),
     endorsed_external_weight: float (default 1.0),
     studied_external_weight: float (default 0.5),
     opposing_external_weight: float (default 0.1) }
   ```
   The default mix: pull from proprietary + endorsed, with
   proprietary weighted 2x. Studied/opposing must be explicitly
   opted-in.

E. Oracle UI checkboxes. The Oracle page renders four checkboxes
   plus the weighting sliders (advanced view, collapsed by
   default). Each checkbox shows a count of available sources of
   that provenance. Founders see exactly what they're querying.

F. Synthesis weighting. The synthesizer engine (built in
   prompt 10) consumes the same provenance_filter. Memos
   generated by the synthesizer record which provenance kinds
   were active and with what weights so a memo's reasoning is
   auditable.

G. Contradiction engine integration. The contradiction engine
   (prompt 06) gains a `provenance_policy` parameter:
   - By default: contradiction tests run within proprietary and
     between proprietary and ENDORSED_EXTERNAL.
   - STUDIED_EXTERNAL and OPPOSING_EXTERNAL are NOT tested for
     contradictions against proprietary — those are EXPECTED to
     differ. Including them would flood the queue.
   - Operator can override via the cost-monitor surface from
     prompt 07.

H. Library page. New public `/library` page (or extend the
   existing one if present) renders endorsed-external sources
   prominently. Studied/opposing-external are listed in a
   collapsed section. Proprietary material remains where it
   already surfaces — Currents, Principles, Articles.

I. Migration plan for existing rows:
   - Every existing artifact gets `provenance = PROPRIETARY` by
     default. Founder reviews via a one-time triage flow at
     `/(authed)/library/triage` and re-tags any rows that should
     be ENDORSED_EXTERNAL etc.
   - The migration includes a count-summary so the founder sees
     "N artifacts defaulted to PROPRIETARY; review them here."

J. CLI:
   - `noosphere library tag --artifact <id> --provenance
       PROPRIETARY|ENDORSED_EXTERNAL|...`
   - `noosphere library list --provenance ENDORSED_EXTERNAL`
   - `noosphere library audit-untagged` — surfaces rows that
     still need triage.

K. Tests:
   - Upload with each provenance choice persists correctly.
   - Provenance propagates to derived claims/principles.
   - Oracle filter includes/excludes correctly per checkbox.
   - Synthesis weights are applied in the order they're declared.
   - Contradiction engine respects provenance_policy by default;
     override works.
   - Migration backfills PROPRIETARY on existing rows.
   - The agent cannot change provenance without an explicit
     CLI/UI action (no inference, no auto-update).
