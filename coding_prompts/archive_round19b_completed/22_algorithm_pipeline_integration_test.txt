Goal: a single dense integration test that exercises the full
Round-19 pipeline end-to-end — principle ingest through algorithm
drafting, runtime, invocation, memo build, portfolio agent
dispatch, paper bet creation, resolution, calibration update,
contradiction-engine inclusion. If THIS test passes, the
algorithm layer is functionally alive. If any one step is
broken, this test catches it before sync.

Per-prompt tests in Round 19 are unit-scale; the smoke harness
from prompt 21 hits surfaces. This prompt's test verifies the
CONTINUITY between layers — the place where Round 19 most
plausibly breaks is at the seams between modules.

The integration test models the founder's arms-race example from
the meeting transcript. End-to-end:

```
[seed] 3 principles about security dilemma + escalation dynamics
   ↓
[seed] 2 currents events with elevated bilateral spending +
        elevated rhetoric escalation index
   ↓
algorithm drafter runs against the 3-principle cluster
   → DRAFT algorithm "Arms-Race Escalation Predictor"
   ↓
[simulated founder] accept the draft → ACTIVE
   ↓
runtime tick → trigger predicate matches → reasoning chain
   executes → AlgorithmInvocation created with reasoning trace
   ↓
synthesizer enqueues from algorithm fire → constitutes question
   → reasoning chain → InvestmentMemo built
   ↓
memo dispatched to a HUMAN portfolio agent → MemoDispatch PENDING
   ↓
[simulated operator] accept-and-bet → PAPER bet on Polymarket
   fixture market
   ↓
[time advance] market resolves YES
   ↓
resolution tracker → ForecastResolution created → algorithm
   calibration updated → BetSpec resolution
   ↓
contradiction-engine check: the algorithm's cited principles
   pass the canonical detector vs. the firm corpus
```

If every step persists the expected rows AND emits the expected
structured logs AND no module raises, the test passes. Any
silent abstain at a step that should have fired = failure.

Constraints
- The test is hermetic: no external HTTP, no real LLM, no real
  market data. Every external dependency is mocked.
- The test is FAIRLY representative — it doesn't mock so much
  that the test passes while the production path would fail.
- A failing integration test surfaces the SPECIFIC stage that
  broke, not "everything is wrong."
- The test uses public APIs of each module; it does NOT poke at
  private fields to make assertions. If a needed assertion
  requires a private field, that's a sign the module needs a
  better public read API.
