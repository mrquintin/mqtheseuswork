from dataclasses import dataclass
from noosphere.models import Conclusion, Claim, ForecastMarket

@dataclass
class RetrievedSource:
    source_type: Literal["CONCLUSION", "CLAIM"]
    source_id: str
    text: str
    relevance: float
    surfaceable: bool      # public-facing safe?
    visibility: Literal["PUBLIC", "FOUNDER", "INTERNAL"]
    metadata: dict          # discipline, confidence, citations chain, etc.

def build_query_from_market(market: ForecastMarket) -> str:
    """
    Compose a single query string the embedding-side retriever can use.

    Recipe:
      title
      "\n\n"
      description (truncated to 600 chars)
      "\n\n"
      "Resolution criteria: " + resolutionCriteria (truncated to 400 chars)
      "\n\n"
      "Category: " + category
    """

def retrieve_for_market(
    store, market: ForecastMarket, *, top_k: int = 8,
) -> list[RetrievedSource]:
    """
    Wrap HybridRetriever.

    Filter rules:
      - drop sources with `visibility != PUBLIC` UNLESS source.surfaceable is True
      - drop revoked conclusions
      - drop sources older than 18 months unless `is_load_bearing` is True
      - apply MMR (maximal marginal relevance) with lambda=0.7 to diversify
      - return at most top_k sources

    On error: never raise. Return [] and emit a `forecasts.retrieval.error`
    structured log. The generator (prompt 06) treats [] as
    INSUFFICIENT_SOURCES.
    """

Reuse — do not rewrite — the underlying retriever. The only Forecasts-specific code is `build_query_from_market` and the post-retrieval filter pass.

### Why a wrapper at all (document this in the module docstring)

> "The Forecasts pipeline retrieves from the same Noosphere index as Currents, but its inputs and stricter surfacing rules differ. Centralizing this wrapper means a future change to Currents-side retrieval cannot silently weaken Forecasts-side surfacing — the filter is enforced here, post-retrieval, where Forecasts owns it."
