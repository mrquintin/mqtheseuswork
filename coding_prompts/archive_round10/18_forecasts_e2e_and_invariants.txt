# Codex Prompt 18 of 18 — End-to-end pipeline test + 8 invariants + Playwright smoke + RELEASE_CHECKLIST

**Sequence:** prompt 18 of 18 (Round 10, Wave I — regression). Depends on 02–17. Report back with: pass/fail of each invariant, the Playwright run video summary, and the contents of `RELEASE_CHECKLIST.md`.

**Why this prompt exists.** Without a regression suite that hits the whole pipeline, future rounds will silently break the safety properties that justify shipping live trading capability at all. This prompt defines and enforces the eight invariants that must hold before, and after, any change to the Forecasts module.

**The eight invariants (these are the contract):**

1. **No prediction without ≥3 sources.** Empty Noosphere + new market → ABSTAINED_INSUFFICIENT_SOURCES; no LLM call. With 1 or 2 sources → same. With 3+ → predict.
2. **Citations are verbatim-anchored.** Every `quoted_span` is a real substring of its cited source. Two failures in retries → ABSTAINED_CITATION_FABRICATION.
3. **Follow-up re-retrieves.** Each user question runs fresh retrieval; the LLM is not given the prediction's saved citations as its only source pool.
4. **Budget enforcement.** Hour-bounded ceilings hold across container restarts (persisted JSON, atomic writes). The Forecasts budget is independent of the Currents budget — exhausting one does not block the other.
5. **Live trading is OFF by default.** Default env produces `mode=PAPER_ONLY` on import. Setting only `FORECASTS_LIVE_TRADING_ENABLED=true` without credentials → `mode=LIVE_DISABLED_NO_CREDENTIALS`.
6. **Live bets require eight gates.** All eight from `safety.py:check_all_gates` must pass; failing any gate refuses the bet and emits a structured-log line with the gate code.
7. **Resolution is append-only.** A second `put_forecast_resolution` for the same predictionId is a no-op (or raises); calibration metrics never decrement on re-runs.
8. **Revoked source propagates.** Revoking a Conclusion that a published prediction cited → that prediction's `revoked_sources_count` becomes ≥ 1 within one scheduler tick.
