   - CORE — ships in the template.
   - FIRM — stays in the source repo, removed from the
     template.
   - CONFIG — ships as a parameterised stub.
   - SEED — ships as an empty-by-default fixture the new

B. Template extraction script
   `scripts/build_template.sh`:
   - Outputs a `theseus-template/` sibling directory
     (gitignored from the SOURCE repo; the script also
     `git init`s the output as a clean, separate repo with
     its own initial commit).
   - Copies every CORE file verbatim.
   - For CONFIG files, runs a token-substitution pass.
     A central manifest `scripts/template/manifest.yml`
     names every replaceable token:
     `THESEUS_ORG_NAME → "<your firm name>"`,
     `THESEUS_DEFAULT_LLM → "haiku-4-5"` (overridable),
     ... etc.
   - For SEED files, ships an empty-shape version (Markdown
     headings, no content; SQL fixtures with one demo row).
   - Strips:
     - the founder's name and bio from anywhere they appear,
     - the firm's research benchmarks (the "Quintin
       Hypothesis" benchmark, etc.) — those are firm-specific
       intellectual property,
     - all `.env*` files,
     - all `coding_prompts/archive_round*` directories,
     - all `.codex_runs/`, `.claude_code_runs/` log
       directories,
     - all generated PDFs except the guides from prompt 67
       (which are general-purpose).

C. Per-tenant boot wizard
   `theseus-template/scripts/bootstrap.sh`:
   - Asks (interactive prompts) for the tenant's:

     - primary admin email,
