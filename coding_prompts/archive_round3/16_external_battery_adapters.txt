## Step-by-step
1. Create an adapter class implementing the Protocol: `name`, `license`, `fetch`, `iter_items`, `resolve`.
2. `fetch` reads the snapshot directory, computes a deterministic content hash over the snapshot files, and returns a `CorpusBundle`.
3. `iter_items` parses the snapshot format and yields `ExternalItem`s with correct `as_of`, `resolved_at`, and `outcome_type`.
4. `resolve` returns an `Outcome` with a concrete `resolution_source` (the corpus's own resolution authority; NEVER the firm).
5. Create a small fixture snapshot under `tests/fixtures/adapters/<corpus>/` with at least 5 items covering the corpus's resolution variations.
6. Write the adapter's tests.
7. Append to `docs/eval/external_corpora.md` a section with: corpus name, license, URL, sample size, outcome kinds supported.

ADAPTER SPECIFICS
gjp.py (Good Judgment Project / Good Judgment Open):
  license = LicenseTag.gjp_public
  Snapshot format: JSON with records { question_id, question_text, opened_at, closed_at, outcome (yes/no/numeric range), gjp_baseline, superforecaster_baseline }.
  iter_items: one ExternalItem per question. outcome_type = binary or interval depending on record.
  resolve: outcome_source = "Good Judgment Project".

metaculus.py:
  license = LicenseTag.metaculus_public
  Snapshot: JSON array of public Metaculus questions with community prediction history and resolution.
  iter_items: one per resolved question (exclude unresolved).
  resolve: outcome_source = "Metaculus public resolution".

claim_review.py:
  license = LicenseTag.claim_review
  Snapshot: JSONL, one per ClaimReview-tagged fact-check.
  iter_items: ExternalItem with question_text = the claim text, outcome_type = binary, resolved_at = fact-check date.
  resolve: outcome_source = fact-checker name from record.

replication.py:
  license = LicenseTag.replication_public
  Snapshot: CSV with columns { study_id, original_effect, replication_effect, replication_source, replication_date }.
  iter_items: one ExternalItem per study. outcome_type = interval (effect size within a CI).
  resolve: outcome_source = replication project name from the row.

## Tests
For each adapter:
1. isinstance check against the Protocol (use `runtime_checkable`).
2. `fetch` is deterministic: running twice against the same snapshot produces identical content_hash.
3. `iter_items` yields exactly the number of items in the fixture; every item has non-empty `question_text` and valid `as_of`.
4. `resolve` returns a non-None Outcome with a non-empty `resolution_source`.
