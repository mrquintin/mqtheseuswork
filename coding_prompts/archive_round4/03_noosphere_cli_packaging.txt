## Your task

You are an implementation agent. Your job is to make the Noosphere CLI packageable as a standalone executable via PyInstaller. Users should be able to install a single `noosphere` binary (or app directory) and run `noosphere ingest`, `noosphere ask`, etc. without needing Python installed.

Key challenges:
- Noosphere has heavy ML dependencies (sentence-transformers, torch, scipy, scikit-learn, spacy, transformers).
- It uses Alembic for database migrations (migration scripts must be bundled).
- It uses a data directory for the knowledge graph, embeddings, and SQLite database.
- Some modules discover plugins or load models at runtime.

When you finish, verify the spec file syntax and the path resolver. Do not proceed to any other work.

## Step-by-step

1. **Create `noosphere/noosphere/frozen_support.py`** — centralized path resolution for frozen mode:
   ```python
   import sys
   import os
   from pathlib import Path

   def is_frozen() -> bool:
       return getattr(sys, 'frozen', False) and hasattr(sys, '_MEIPASS')

   def bundle_dir() -> Path:
       """Root of the PyInstaller bundle (or the source tree in dev)."""
       if is_frozen():
           return Path(sys._MEIPASS)
       return Path(__file__).resolve().parent.parent

   def data_dir() -> Path:
       """User-writable directory for Noosphere data (graph, DB, embeddings).
       Respects THESEUS_DATA_DIR env var, else uses platform default."""
       env = os.environ.get("THESEUS_DATA_DIR")
       if env:
           d = Path(env)
       elif sys.platform == "darwin":
           d = Path.home() / "Library" / "Application Support" / "Noosphere"
       elif sys.platform == "win32":
           d = Path(os.environ.get("APPDATA", Path.home())) / "Noosphere"
       else:
           d = Path.home() / ".noosphere"
       d.mkdir(parents=True, exist_ok=True)
       return d

   def alembic_dir() -> Path:
       """Location of Alembic migration scripts — inside the bundle when frozen."""
       return bundle_dir() / "alembic"

   def alembic_ini() -> Path:
       return bundle_dir() / "alembic.ini"
   ```

4. **Create `noosphere/noosphere.spec`** — the PyInstaller spec file:
   - Entry point: `noosphere/noosphere/__main__.py`.
   - `datas`:
     - `('alembic', 'alembic')` — bundle migration scripts
     - `('alembic.ini', '.')` — bundle alembic config
     - Sentence-transformers model cache is NOT bundled (too large) — models download on first run. Document this in a comment.
   - `hiddenimports`: `typer`, `rich`, `click`, `anthropic`, `openai`, `sqlmodel`, `sqlalchemy`, `alembic`, `pydantic`, `pydantic_settings`, `structlog`, `sentence_transformers`, `torch`, `numpy`, `scipy`, `sklearn`, `spacy`, `networkx`, `hdbscan`, `umap`, `transformers`, `pynacl`, `zstandard`, `yaml`, `matplotlib`. Also add `noosphere.noosphere.*` submodules and `noosphere.noosphere.coherence.*`, `noosphere.noosphere.methods.*`, `noosphere.noosphere.cascade.*`, etc. (all subpackages). Use a collect approach:
     ```python
     from PyInstaller.utils.hooks import collect_submodules
     hiddenimports = collect_submodules('noosphere.noosphere')
     hiddenimports += collect_submodules('sentence_transformers')
     hiddenimports += [...]
     ```
   - `excludes`: `tensorflow`, `tensorboard`, `jupyter`, `IPython`, `pytest`, `matplotlib.backends` (keep core matplotlib for any plotting).
   - `runtime_hooks`: `['scripts/build_hooks/hook-noosphere.py']`.
   - `console=True` (this IS a CLI tool — it needs a terminal).
   - Name: `noosphere`.
   - Single-directory mode (NOT onefile — torch is too large).
   - On macOS, do NOT create a `.app` bundle (it's a CLI, not a GUI).
