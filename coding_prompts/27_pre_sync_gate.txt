Implement:

A. `scripts/ready-to-sync.sh`:
   - One-line invocation, no arguments by default.
   - Sequence of subcommands, one per gate step:
     - `scripts/check_migration_linearity.py`
     - `scripts/check_no_import_cycles.py && scripts/generate_api_types.py --check`
     - `scripts/smoke/run.sh`
     - `pytest tests/integration -m integration -q`
     - `noosphere env validate --mode full`
     - `pytest tests/safety -q`
     - `pytest tests/regression -q`
     - `scripts/check_ci_workflow_integrity.py && scripts/check_doc_freshness.py`
   - On any step failure: prints a clear "Gate FAILED at step N:
     <step name>. See <log path>" message and exits non-zero.
   - On success: prints a single "Gate PASSED. Safe to sync."
     line with a green check mark in tty mode.
   - Supports a `--from <step>` flag for resuming after a fix
     (operator fixes step 3 → re-runs the gate from step 3
     forward).
   - Supports a `--only <step>` flag.
   - Supports `--skip <step>` for genuine bypass (logs the skip
     prominently; the sync script respects the skip but warns
     loudly on push).
   - Final report saved to
     `docs/verification/ready_to_sync/<timestamp>/REPORT.md`.

B. `scripts/sync-to-github.sh` extension:
   - At the top of the sync flow (after the pre-commit hook but
     before the actual push), invoke `scripts/ready-to-sync.sh`.
   - If it fails: sync refuses. Prints the gate's failure
     report path and the resume command.
   - `--skip-ready-to-sync` is the explicit bypass (logs a
     warning AND records the skip in
     `docs/verification/ready_to_sync_skips.log` for audit).
   - `--ready-to-sync-only` runs only the gate (no push).
     Useful for the operator who wants to verify before
     committing.
