# Codex Prompt 1 of 5 — Credential validator (read-only, never echoes secrets)

**Sequence:** prompt 1 of 5 (Round 11, Wave A — pre-flight). Depends on Round 10's full implementation. Report back with: a green/red status table for each external service, the masked summary that proves every key is *present* but does not print any key value.

**Why this prompt exists.** Live trading is a footgun pointed at money. Before a single live order can be submitted, the operator must know which credentials are actually working. A typo in `.env.live` should fail fast with a precise pointer ("Kalshi PEM is malformed at line 3"), not silently halve the system's coverage and show up days later as "why is Kalshi never resolving anything?". The validator runs every external read-only path the system depends on, reports per-service status, and never prints, logs, or otherwise discloses any secret value.

**Before you start — read current state:**
1. Read `.env.live.template` at the repo root — the canonical list of env vars. Anything you validate must come from this set; do not invent new ones.
2. Read `noosphere/noosphere/forecasts/config.py` — the `PolymarketConfig` and `KalshiConfig` dataclasses already wrap env-var loading. Reuse them; do not parse the env directly.
3. Read `noosphere/noosphere/forecasts/_kalshi_client.py` — the RSA signing flow you will call.
4. Read `noosphere/noosphere/llm.py` (or wherever `ANTHROPIC_API_KEY` is loaded) for the existing client.

## Your task

### `noosphere/scripts/validate_live_credentials.py`

```python
#!/usr/bin/env python3
"""
Validate every credential required for live trading by performing a single
read-only call against each external service.

USAGE: python -m noosphere.scripts.validate_live_credentials [--strict]
       --strict: exit non-zero if any optional service is missing.

The script never prints, logs, or returns any secret value. Each check
either succeeds (green ✓) or fails with a redacted reason (red ✗).
"""
```

The validator runs these checks in sequence, each with a 10s timeout:

| # | Service | Check |
|---|---|---|
| 1 | `DATABASE_URL` | Connect via psycopg/SQLAlchemy, run `SELECT 1`, count `ForecastMarket` rows |
| 2 | `ANTHROPIC_API_KEY` | One Haiku 4.5 call requesting `max_tokens=1`. Any 200 with content is success |
| 3 | `POLYMARKET_GAMMA_BASE` | `GET /markets?limit=1`. Any 200 with a JSON list is success (no auth required) |
| 4 | `POLYMARKET_PRIVATE_KEY` | If unset → "OPTIONAL — paper-only mode". If set: derive the wallet address via `eth_account` and report only the address (NOT the key) |
| 5 | `KALSHI_API_KEY_ID` + `KALSHI_API_PRIVATE_KEY` | Sign a request, hit `GET /exchange/announcements` (lightest auth-required endpoint), confirm 200 |
| 6 | `KALSHI_DEMO_*` | Same as above against the demo base URL. Required if `FORECASTS_LIVE_TRADING_ENABLED` is `true` and `POLYMARKET_PRIVATE_KEY` is set; otherwise informational |
| 7 | `FORECASTS_OPERATOR_SECRET` | Hex-decode + length check (must be ≥ 32 bytes after decode) |
| 8 | All other ceiling/budget vars | Parse to expected types; flag if `FORECASTS_LIVE_TRADING_ENABLED=true` but any of the ceilings is `0` |

### Output format

The script prints a single-block status table to stdout, then a final
"all green" or "X failures" summary.

```
Theseus Forecasts — credential validation

  ✓  Database              postgresql://****:****@****/postgres        128 ForecastMarket rows
  ✓  Anthropic             sk-ant-***                                  Haiku 4.5 reachable, 1 token
  ✓  Polymarket Gamma      gamma-api.polymarket.com                    20-row sample fetched
  ✓  Polymarket wallet     0x9f3a…7cE1                                 (derived from key; key never printed)
  ✓  Kalshi (live)         api.elections.kalshi.com                    auth ok
  ✗  Kalshi (demo)         (not configured)                            REQUIRED for the rehearsal flow
  ✓  Operator HMAC secret  (32-byte secret present)
  ✓  Trading mode          PAPER_ONLY                                  live disabled (correct default)

  RESULT: 7/8 green. The demo Kalshi keys are required for prompt 03's
  testnet rehearsal — populate them in .env.live before continuing.
```

**Hard rule:** the validator never prints a key, a PEM, a database password, or a wallet private key. If a check needs to display a value, it shows a structural mask (`sk-ant-***`, `0x9f3a…7cE1`, `(32-byte secret present)`) — the FULL value never crosses stdout, stderr, or any log line.

### Per-check redaction tests

`noosphere/tests/test_validate_live_credentials.py`:
- `test_no_secret_in_stdout`: monkeypatch every external call to a mock; populate test `.env` with marker values like `POLYMARKET_PRIVATE_KEY=0xMARKERVALUE_DO_NOT_LEAK`; run the validator; assert `MARKERVALUE_DO_NOT_LEAK` does not appear anywhere in captured stdout/stderr.
- `test_no_secret_in_traceback`: arrange a forced exception inside the Kalshi check; assert the resulting traceback string contains no key bytes.
- `test_paper_only_path`: with `POLYMARKET_PRIVATE_KEY` unset, the validator marks Polymarket-wallet as "OPTIONAL" and the suite still reports green overall (when not `--strict`).
- `test_strict_mode`: same setup with `--strict`; the validator returns exit code 2.
- `test_database_count_query`: a row-count discrepancy is informational only, not a failure.

### CLI integration

Add a Typer subcommand `noosphere forecasts validate-credentials` that calls the script with the same flags. This is the canonical operator entry point.

## Definition of done

- `python -m noosphere.scripts.validate_live_credentials` runs against an env-populated machine and produces the table above.
- The redaction tests pass — verified by `grep`-ing captured output for the marker strings.
- The validator does NOT mutate any DB row, place any order, or send any non-read-only API call.
- The Anthropic check uses `max_tokens=1` so the cost is negligible (a few hundred input tokens, one output token).
