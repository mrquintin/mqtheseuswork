# Codex Prompt 5 of 5 — Deployment scripts + minimal observability for live trading

**Sequence:** prompt 5 of 5 (Round 11, Wave C — operations). Depends on 01–04. Report back with: the deployment manifest, the metrics endpoints exposed, and the alerting rules wired.

**Why this prompt exists.** Live trading is unattended for most hours of the day. The system needs to fail visibly: an operator must learn within minutes — not the next morning — when the scheduler is down, the kill switch has auto-engaged, the budget is exhausted, or a live bet has failed. This prompt ships the deployment glue (Vercel for the frontend, a single VM via systemd or docker-compose for the API + scheduler) and a minimum-viable observability layer that is loud enough to be useful and quiet enough to not alarm-fatigue.

**Before you start — read current state:**
1. Read `Dockerfile.api`, `Dockerfile.scheduler`, `docker-compose.yml` — Round 10 prompt 20 already ships container glue. Extend it; do not replace.
2. Read `current_events_api/current_events_api/metrics.py` — Prometheus text endpoint already exists for Currents. Extend with Forecasts metrics.
3. Read `noosphere/noosphere/forecasts/status.py` — the status JSON written every scheduler tick. The healthcheck reads from this.

`current_events_api/current_events_api/metrics.py` — extend with these Forecasts counters (Prometheus text format):

### 4. Alerting rules

`deploy/alerts/forecasts.alerts.yml` — Prometheus alerting rules. (The user wires this to Alertmanager + Slack/email themselves; the file documents the alert intent.)

Rules at minimum:
- **kill_switch_engaged**: `theseus_forecasts_kill_switch_engaged == 1` for 1m → page operator. Severity: critical.
- **scheduler_stale**: `time() - theseus_forecasts_last_tick_ts > 1800` for 5m → page operator. The scheduler hasn't ticked in 30 min.
- **brier_drift**: `theseus_forecasts_brier_mean_30d > 0.30` for 24h → email operator (not page; this is a slow-developing problem). Calibration has degraded.
- **live_bet_failed_streak**: `increase(theseus_forecasts_live_bet_status_total{status="FAILED"}[15m]) >= 3` → page operator. Three live failures in 15 min — likely an exchange or signing issue.
- **budget_exhausted**: `theseus_forecasts_budget_charged_pct > 0.95` for 5m → email operator.
