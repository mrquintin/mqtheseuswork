   - Shared API payloads (cross-language) → JSON Schema, generated
     to both Pydantic and TS.

   `packages/shared-schemas/` as JSON Schema files. Generate

   the build.

   were hand-written separately in Round 17, replace one side with

   the hand-written TS interface with `Prisma.X`.

   (noosphere ↔ codex via API or shared DB), validate payloads
   against the JSON schema. Validation failure logs and surfaces in
   the unified attention queue (Round 17 prompt 34) rather than
   silently drifting.

E. Tests. A "type contract" test that imports both the Pydantic
   model and the TS type for each shared schema and round-trips a
   golden fixture through both.
