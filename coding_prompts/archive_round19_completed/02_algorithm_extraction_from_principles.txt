Goal: an LLM-assisted drafter that proposes LogicalAlgorithms from
clusters of related principles. Output is structured (matches the
schema from prompt 01). Drafts go into a founder triage queue;
the founder accepts, edits, or rejects. Never auto-promotes to
ACTIVE.

The drafter's job is the founder's hardest job done at draft
quality: looking at a cluster of related principles and asking
"what observable inputs would these jointly let us predict
outputs for?" The drafter produces candidates the founder either
sharpens or discards.

What the drafter SHOULD propose — concrete shape:

Cluster: [
  "States in mutual threat perception engage in security-dilemma
   escalation absent credible commitment devices.",
  "Domestic political incentives reinforce external escalation once
   initiated; reversal requires elite cost.",
  "Arms races are predicted by the second derivative of military
   spending, not the first."
]
→ Drafter proposes "Arms-Race Escalation Predictor" algorithm with
  inputs (spending YoY delta per side, rhetoric escalation index,
  mediator presence), reasoning chain (detect → apply each
  principle → synthesize), output (per-side projected spending
  increase, horizon, confidence band).

What the drafter MUST REFUSE:

- Cluster of principles that are normative-only (no observable
  inputs) → drafter emits UNFORMALISABLE with a reason.
- Cluster too small (< 2 principles) → defers.
- Cluster whose principles have already produced an ACTIVE
  algorithm in the last 30 days → flagged as duplicate, awaiting
  founder review.
- Principles whose `domain_of_applicability` does not overlap →
  drafter does NOT manufacture a cross-domain algorithm. It
  emits NO_DOMAIN_OVERLAP.

A. `noosphere/noosphere/algorithms/drafter.py`:
   - `class AlgorithmDrafter`:
     - `async draft_from_cluster(store, principle_ids, *,
                                  budget) -> DraftResult`:
       1. Load the principles. Validate cluster size and domain
          overlap.
       2. Compose LLM prompt with the cluster + the three
          worked examples from this prompt's preamble (the
          arms-race / hyperstition / founder-quality shapes)

       3. Demand strict JSON output matching
          LogicalAlgorithm's structure (inputs, output, reasoning

       4. Validate the trigger predicate via the validator

       6. Validate that every input has an `observability_source`
          that is either a known Currents/Upload provider
          identifier OR explicitly marked "manual.operator.entered".
       7. Persist as DRAFT.
   - `class DraftResult`:
       outcome ∈ DRAFTED | UNFORMALISABLE | NO_DOMAIN_OVERLAP |
                  CLUSTER_TOO_SMALL | DUPLICATE_RECENT |
                  ABSTAINED_BUDGET | ABSTAINED_FABRICATION
       algorithm_id (nullable)
       reason (text)

Constraints
- No auto-promotion. The agent NEVER moves an algorithm to ACTIVE.
- The drafter is bounded by budget; over-budget runs emit

- The drafter does not modify principles. It only reads them.
- Drafts that fail validation are NOT persisted as drafts —
  they're abandoned. Founders should never see broken drafts.
