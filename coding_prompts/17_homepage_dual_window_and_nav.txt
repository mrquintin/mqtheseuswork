## Your task

### 1. Homepage `<DualPulseSection>` component

`theseus-codex/src/app/(home)/DualPulseSection.tsx` (or wherever the homepage components live):

```
+------------------------------------------------------+------------------------------------------------------+
|  ⏵ CURRENTS — live opinion                           |  ◇ FORECASTS — live predictions                       |
|  ----------------------------------------------------|  ----------------------------------------------------|
|  <OpinionCard>                                        |  <ForecastCard>                                       |
|  <OpinionCard>                                        |  <ForecastCard>                                       |
|  <OpinionCard>                                        |  <ForecastCard>                                       |
|  <OpinionCard>                                        |  <ForecastCard>                                       |
|                                                       |                                                        |
|  view all currents →                                  |  view all forecasts →                                  |
+------------------------------------------------------+------------------------------------------------------+
```

- Server-renders the most recent 4 of each from `/api/currents?limit=4&seeded=true` and `/api/forecasts?limit=4&seeded=true`.
- Hydrates with `useLiveOpinions` and `useLiveForecasts` so new items stream in to either side independently.
- Each side has its own LivePulse indicator.
- Section header tagline (verbatim or close): "Theseus thinks out loud — what's happening, and what's about to."

#### Responsive behavior

- ≥1024px: two columns side-by-side, equal width, with a subtle vertical rule between.
- 720–1023px: two columns, narrower padding, the rule kept.
- <720px: stacked vertically; CURRENTS first (it has the bigger volume of activity and reads first to a returning user); a tab toggle at the top of the stacked layout lets the user pick which to show full-height.

#### Empty / disconnected states

- If one side has no data: render a parchment block with the empty message and keep the other side live.
- If both sides have no data: render a single full-width block with "Theseus is starting up — first opinion expected within the hour."
