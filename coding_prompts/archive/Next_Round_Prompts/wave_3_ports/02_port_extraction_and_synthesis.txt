NEW METHOD — method_candidate_extractor (NOT a port)
═══════════════════════════════════════════════════════════════════════════════
Create `methods/extract_from_corpus.py` containing a new registered method `method_candidate_extractor`:
- method_type = EXTRACTION
- version = "1.0.0"
- nondeterministic = True
- Input: a pydantic `MethodExtractionInput { artifact_refs: list[str], window_chars: int = 2000 }`
- Output: a pydantic `MethodExtractionOutput { candidates: list[MethodCandidate] }`
- Behavior: scan each artifact for chunks matching any of a small regex catalog ("the way to tell", "the test we use", "the rule of thumb", "we determine X by") AND pass candidate chunks through an LLM with a structured prompt that extracts `{ name, description, rationale, preconditions, postconditions }`.
- A `MethodCandidate` pydantic model is defined at the top of this file (not in `models.py`).
- Each candidate is persisted via a new `Store.insert_method_candidate` helper — if this helper does not exist yet in Store, treat it as a TODO: create a row in `method_candidates` table using raw SQLModel here, and flag to the orchestrator that the Store helper is missing (write a `WAVE3_TODO_STORE_HELPER.md` note at the repo root listing the missing method).
