STEP-BY-STEP
═══════════════════════════════════════════════════════════════════════════════
1. Build `routes/round3.py` exposing READ-ONLY endpoints:
     GET /methods
     GET /methods/{name}/{version}
     GET /methods/{name}/{version}/doc
     GET /methods/{name}/{version}/eval-card
     GET /conclusions/{id}/provenance

     GET /methodology/rigor
     GET /methodology/overrides
     GET /methodology/decay
     GET /interop
     GET /interop/{name}/{version}
   Every endpoint returns a pydantic response model. No mutation endpoints on this surface.
2. Modify `app.py` to include the new router.
3. In `packages/theseus-api-types/`, create a small TS package publishing interfaces for every response model. Use a Python script (or manually mirror) to ensure types stay in sync. Add a build step `tsc --declaration`.
4. Write `tests/test_round3_routes.py` exercising every endpoint with a seeded test store.
5. Both sibling agents' `lib/api/round3.ts` should import these shared types.
