STEP-BY-STEP
═══════════════════════════════════════════════════════════════════════════════
1. Build `signing.py` reusing `ledger.keys.KeyRing` to sign CHECKSUMS files and packaged artifacts.
2. Build `adapter_template.py` which emits a string template for a packaged method's `adapter.py`. The template must import ONLY from the packaged implementation and PyPI — never from `noosphere.*`.
3. Build `docker_build.py` which emits a `Dockerfile` pinning the Python version, system deps, and the packaged code, and runs `docker build` and captures the image digest.
4. Build `package_method.py` with `package(method_ref: MethodRef, out_dir: Path) -> PackagePath`. It must produce in `out_dir`:
     method.json           (Method spec serialized)
     rationale.md          (copied from methods/<name>.RATIONALE.md)
     implementation/       (self-contained copy of implementation code and its pinned deps)
     adapter.py            (generated from adapter_template)
     Dockerfile
     CHECKSUMS             (SHA256 of every file, signed)
     README.md             (how to run, input/output schema, known limitations)
     EVAL_CARD.md          (populated from CounterfactualEvalRun + BatteryRunResult rows if present; else a "none yet" stub)
     LICENSE               (default Apache 2.0)
5. Build `harness.py` with `run_transfer_study(method_ref, source_domain, target_domain, dataset) -> TransferStudy`, decorated as a registered method (name="transfer_study").
6. Build `scripts/check_packaging_selfcontainment.py`: launch a subprocess that sets PYTHONPATH to exclude `noosphere.*`, imports `adapter.py`, asserts no ImportError. Fail CI on violation.
7. Build `scripts/check_mip_versioning.py`: scan git log for commits that change implementation files of a method WITHOUT bumping `Method.version`. Fail on violation.
8. Write all tests.
9. `pytest noosphere/tests/test_transfer_*.py -v`.
