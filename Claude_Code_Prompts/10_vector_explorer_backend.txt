## The problem

The founder wants to "explore the ideas and questions that they have transcribed and put into the codex in the vector embedding space on a set of axes." This means:

1. Conclusions need vector embeddings (they may already have them in Noosphere, but they need to be accessible from the Codex).
2. The system needs to compute meaningful semantic axes — dimensions along which conclusions can be plotted.
3. An API endpoint needs to serve the projected coordinates so the frontend can render a visualization.

This prompt handles the backend: embedding storage, axis computation, and the API endpoint. Prompt 11 handles the frontend visualization.

2. **Create `theseus-codex/src/lib/embeddingAxes.ts`** — a utility that takes an array of embedding vectors and computes meaningful axes:

   ```typescript
   /**
    * Compute 2D or 3D projection axes for a set of conclusion embeddings.
    *
    * Approach: PCA (Principal Component Analysis) on the embedding matrix.
    * The top 2–3 principal components become the axes. Each axis is labeled
    * by examining which conclusions load most heavily on it and generating
    * a descriptive label.
    */

   export interface ProjectedConclusion {
     id: string;
     text: string;
     x: number;
     y: number;
     z?: number;
     topicHint: string;
     confidenceTier: string;
   }

   export interface SemanticAxis {
     index: number;
     label: string;           // e.g., "Concrete ↔ Abstract"
     varianceExplained: number;
     positivePole: string[];  // sample conclusion texts at the positive end
     negativePole: string[];  // sample conclusion texts at the negative end
   }

   export interface EmbeddingProjection {
     conclusions: ProjectedConclusion[];
     axes: SemanticAxis[];
   }
   ```

   Implement PCA using basic linear algebra (no external library required for 2D projection of relatively small matrices — the org won't have more than a few thousand conclusions):
   - Center the embedding matrix (subtract mean vector).
   - Compute covariance matrix.
   - Extract top 2 eigenvectors via power iteration (sufficient for visualization).
   - Project each conclusion onto the 2D plane.
   - Label each axis by examining the conclusions at the extremes of that axis.

   If the linear algebra is too complex for a from-scratch implementation, use a simpler approach:
   - Import a lightweight library or use the Noosphere Python backend to compute the projection.
   - Create a small API bridge that calls a Python script for the PCA computation.

3. **Create the API route `GET /api/conclusions/embeddings`:**
   - Authenticate via `requireTenantContext()`.
   - Fetch all conclusions with non-null `embeddingJson` for the org.
   - Parse the embedding vectors from JSON.
   - Run the PCA projection.
   - Return the `EmbeddingProjection` response.
   - Cache aggressively: the projection only changes when new conclusions are ingested.

   ```typescript
   import { NextResponse } from "next/server";
   import { requireTenantContext } from "@/lib/tenant";
   import { db } from "@/lib/db";
   import { computeProjection } from "@/lib/embeddingAxes";

   export async function GET() {
     const tenant = await requireTenantContext();
     if (!tenant) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

     const conclusions = await db.conclusion.findMany({
       where: {
         organizationId: tenant.organizationId,
         embeddingJson: { not: null },
       },
       select: {
         id: true,
         text: true,
         topicHint: true,
         confidenceTier: true,
         embeddingJson: true,
       },
     });

     if (conclusions.length < 3) {
       return NextResponse.json({
         conclusions: [],
         axes: [],
         error: "Need at least 3 conclusions with embeddings for visualization",
       });
     }

     const projection = computeProjection(conclusions);
     return NextResponse.json(projection);
   }
   ```
