### 3. FastAPI hosting

Vercel cannot host long-lived Python with SSE. Two supportable targets:

**Option A (recommended):** A single small VM (Fly.io, DigitalOcean droplet, or a Mac mini you already own). Provide a `Dockerfile.api` and a `Dockerfile.scheduler`, plus a `docker-compose.yml` that wires both with a shared `currents-data` volume. Document the deploy in `docs/Vercel_Supabase_Deploy.md`'s Currents section.

**Option B:** Two separate Vercel-friendly serverless deployments using a different runtime (e.g. Modal, Replicate). Less recommended because SSE is awkward and budgets are harder to enforce per-process.

Implement A.

### 4. Dockerfiles

```dockerfile
# Dockerfile.api
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY noosphere ./noosphere
COPY current_events_api ./current_events_api
EXPOSE 8088
CMD ["uvicorn", "current_events_api.main:app", "--host", "0.0.0.0", "--port", "8088", "--workers", "1"]
```

```dockerfile
# Dockerfile.scheduler
FROM python:3.12-slim
WORKDIR /app

```

### 5. `docker-compose.yml` (top-level)

Two services, shared volume, dependent on a Postgres URL from `.env`:

```yaml
services:
  currents-api:
    build: { context: ., dockerfile: Dockerfile.api }
    env_file: ./current_events_api/.env
    ports: ["8088:8088"]
    volumes: ["currents-data:/data/noosphere"]
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request,sys;sys.exit(0 if urllib.request.urlopen('http://localhost:8088/healthz',timeout=2).status==200 else 1)"]
      interval: 15s
      retries: 3
    restart: unless-stopped
  currents-scheduler:
    build: { context: ., dockerfile: Dockerfile.scheduler }
    env_file: ./current_events_api/.env
    volumes: ["currents-data:/data/noosphere"]
    restart: unless-stopped
volumes:
  currents-data:
```
