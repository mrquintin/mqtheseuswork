### 1. Budget persistence

Edit `noosphere/currents/budget.py` so `HourlyBudgetGuard` writes its counters to `NOOSPHERE_DATA_DIR/currents_budget.json` after every `charge()`. The file format:

```json
{
  "window_start_iso": "2026-04-20T14:00:00Z",
  "prompt_tokens": 812345,
  "completion_tokens": 244120,
  "max_prompt_tokens": 1500000,
  "max_completion_tokens": 400000
}
```

On construction, `HourlyBudgetGuard.load(path)` reads the file if present, compares `window_start_iso` to the current UTC hour; if the hour has rolled over, reset counters to zero and update `window_start_iso`. Never carry balances across hour boundaries. Writes are atomic (`tempfile + os.replace`).

`HourlyBudgetGuard.remaining_prompt_tokens()` and `.remaining_completion_tokens()` expose headroom so the scheduler can log them.

If either remaining figure drops below `PER_OPINION_BUDGET_RESERVE = 12_000` prompt tokens / `3_000` completion tokens (two standard opinions' worth), the guard raises `BudgetExhausted` on the next `authorize()` call. Callers in `opinion_generator.py` already handle this exception by returning `OpinionOutcome.ABSTAINED_BUDGET`.

### 2. The scheduler loop

Create `noosphere/currents/scheduler.py`:

```python
from __future__ import annotations

import argparse
import asyncio
import logging
import signal
import time
from dataclasses import dataclass, asdict
from pathlib import Path

from noosphere.store import Store, open_store
from noosphere.currents.x_ingestor import ingest_once, IngestorConfig
from noosphere.currents.enrich import enrich_event
from noosphere.currents.relevance import check_relevance, RelevanceDecision
from noosphere.currents.opinion_generator import generate_opinion, OpinionOutcome
from noosphere.currents.budget import HourlyBudgetGuard, BudgetExhausted
from noosphere.currents.status import write_status

logger = logging.getLogger("currents.scheduler")

CYCLE_SECONDS = 300  # 5 minutes
MAX_EVENTS_PER_CYCLE = 40  # hard cap so one bad cycle cannot drain the budget
MAX_OPINIONS_PER_CYCLE = 12
SHORT_BACKOFF_SECONDS = 30
LONG_BACKOFF_SECONDS = CYCLE_SECONDS  # full cycle skip on budget exhaustion


@dataclass
class CycleReport:
    cycle_id: str
    started_at: str
    duration_ms: int
    ingested: int
    enriched: int
    abstained_insufficient: int
    abstained_near_duplicate: int
    abstained_budget: int
    opined: int
    errors: list[str]
    remaining_prompt_tokens: int
    remaining_completion_tokens: int


async def run_cycle(
    store: Store,
    ingestor_cfg: IngestorConfig,
    budget: HourlyBudgetGuard,
) -> CycleReport:
    from uuid import uuid4
    from datetime import datetime, timezone

    started = time.monotonic()
    started_at = datetime.now(timezone.utc).isoformat()
    cycle_id = f"cyc_{uuid4().hex[:10]}"
    errors: list[str] = []

    # 1. Ingest
    try:
        ingest_report = await ingest_once(store, ingestor_cfg)
        new_event_ids = ingest_report.new_event_ids[:MAX_EVENTS_PER_CYCLE]
    except Exception as e:
        logger.exception("ingest_failed", extra={"cycle_id": cycle_id})
        errors.append(f"ingest:{type(e).__name__}")
        new_event_ids = []

    ingested = len(new_event_ids)
    enriched = 0
    abstained_insufficient = 0
    abstained_near_duplicate = 0
    abstained_budget = 0
    opined = 0

    for event_id in new_event_ids:
        # 2. Enrich
        try:
            enrich_event(store, event_id)
            enriched += 1
        except Exception as e:
            logger.exception("enrich_failed", extra={"cycle_id": cycle_id, "event_id": event_id})
            errors.append(f"enrich:{event_id}:{type(e).__name__}")
            continue

        if opined >= MAX_OPINIONS_PER_CYCLE:
            break

        # 3. Relevance gate
        try:
            decision = check_relevance(store, event_id)
        except Exception as e:
            logger.exception("relevance_failed", extra={"cycle_id": cycle_id, "event_id": event_id})
            errors.append(f"relevance:{event_id}:{type(e).__name__}")
            continue

        if decision == RelevanceDecision.ABSTAIN_NEAR_DUPLICATE:
            abstained_near_duplicate += 1
            continue
        if decision == RelevanceDecision.ABSTAIN_INSUFFICIENT_SOURCES:
            abstained_insufficient += 1
            continue

        # 4. Generate
        try:
            outcome = await generate_opinion(store, event_id, budget=budget)
        except BudgetExhausted:
            abstained_budget += 1
            logger.warning("budget_exhausted_mid_cycle", extra={"cycle_id": cycle_id})
            break
        except Exception as e:
            logger.exception("generate_failed", extra={"cycle_id": cycle_id, "event_id": event_id})
            errors.append(f"generate:{event_id}:{type(e).__name__}")
            continue

        if outcome is OpinionOutcome.PUBLISHED:
            opined += 1
        elif outcome is OpinionOutcome.ABSTAINED_BUDGET:
            abstained_budget += 1
        # other ABSTAINED_* outcomes are counted via the relevance branch

    duration_ms = int((time.monotonic() - started) * 1000)
    report = CycleReport(
        cycle_id=cycle_id,
        started_at=started_at,
        duration_ms=duration_ms,
        ingested=ingested,
        enriched=enriched,
        abstained_insufficient=abstained_insufficient,
        abstained_near_duplicate=abstained_near_duplicate,
        abstained_budget=abstained_budget,
        opined=opined,
        errors=errors,
        remaining_prompt_tokens=budget.remaining_prompt_tokens(),
        remaining_completion_tokens=budget.remaining_completion_tokens(),
    )
    logger.info("cycle_complete", extra={"report": asdict(report)})
    write_status(report)
    return report


async def loop(store: Store, ingestor_cfg: IngestorConfig, budget: HourlyBudgetGuard) -> None:
    stop = asyncio.Event()

    def _handle_sigterm(*_):
        logger.info("shutdown_signal_received")
        stop.set()

    for sig in (signal.SIGTERM, signal.SIGINT):
        try:
            asyncio.get_running_loop().add_signal_handler(sig, _handle_sigterm)
        except NotImplementedError:
            pass  # Windows / non-main thread

    while not stop.is_set():
        t0 = time.monotonic()
        try:
            report = await run_cycle(store, ingestor_cfg, budget)
        except Exception:
            logger.exception("cycle_crashed")
            await asyncio.wait_for(stop.wait(), timeout=SHORT_BACKOFF_SECONDS) if stop.is_set() else await asyncio.sleep(SHORT_BACKOFF_SECONDS)
            continue

        elapsed = time.monotonic() - t0
        sleep_for = max(5.0, CYCLE_SECONDS - elapsed)
        if report.abstained_budget and report.opined == 0:
            sleep_for = LONG_BACKOFF_SECONDS  # back off a full cycle on budget exhaustion
        try:
            await asyncio.wait_for(stop.wait(), timeout=sleep_for)
        except asyncio.TimeoutError:
            pass


def main(argv: list[str] | None = None) -> int:
    parser = argparse.ArgumentParser(prog="noosphere.currents")
    sub = parser.add_subparsers(dest="cmd", required=True)
    sub.add_parser("once", help="Run exactly one pipeline cycle and exit")
    sub.add_parser("loop", help="Run the standing 5-minute scheduler loop")
    args = parser.parse_args(argv)

    logging.basicConfig(
        level=logging.INFO,
        format='{"ts":"%(asctime)s","level":"%(levelname)s","name":"%(name)s","msg":"%(message)s"}',
    )

    store = open_store()  # reads NOOSPHERE_DATA_DIR from env
    ingestor_cfg = IngestorConfig.from_env()
    budget = HourlyBudgetGuard.load(Path(store.data_dir) / "currents_budget.json")

    if args.cmd == "once":
        asyncio.run(run_cycle(store, ingestor_cfg, budget))
        return 0
    if args.cmd == "loop":
        asyncio.run(loop(store, ingestor_cfg, budget))
        return 0
    return 2
```

```python
from __future__ import annotations
import json, os, tempfile
from dataclasses import asdict
from pathlib import Path
from noosphere.store import data_dir  # returns NOOSPHERE_DATA_DIR path

def write_status(report) -> None:
    path = Path(data_dir()) / "currents_status.json"
    payload = {"last_cycle": asdict(report)}
    tmp = tempfile.NamedTemporaryFile("w", dir=path.parent, delete=False)
    try:
        json.dump(payload, tmp, indent=2)
        tmp.flush()
        os.replace(tmp.name, path)
    finally:
        try: os.unlink(tmp.name)
        except FileNotFoundError: pass
```

### 4. The in-process publisher (API side)

`current_events_api` runs in a different process from the scheduler (see prompt 16's compose file). The `OpinionBus` in-memory queue is therefore empty unless something inside the API process puts items on it. Create `noosphere/currents/publisher.py`:

```python
from __future__ import annotations

import asyncio
import logging
from datetime import datetime, timezone
from noosphere.store import Store

logger = logging.getLogger("currents.publisher")

TAIL_INTERVAL_SECONDS = 2.0


class OpinionTailer:
    """Poll the DB for new opinions, push onto the in-process OpinionBus.

    Each API process runs its own tailer; each maintains its own cursor. It is
    deliberately at-least-once for SSE clients — duplicate frames are harmless
    because the client de-dupes by opinion id (see useLiveOpinions in
    theseus-public).
    """

    def __init__(self, store: Store, bus, *, interval: float = TAIL_INTERVAL_SECONDS) -> None:
        self._store = store
        self._bus = bus
        self._interval = interval
        self._last_generated_at: datetime | None = None
        self._task: asyncio.Task | None = None

    async def start(self) -> None:
        # Initialize cursor to "now" so we don't replay historic opinions at startup.
        self._last_generated_at = datetime.now(timezone.utc)
        self._task = asyncio.create_task(self._run(), name="opinion_tailer")

    async def stop(self) -> None:
        if self._task:
            self._task.cancel()
            try: await self._task
            except asyncio.CancelledError: pass

    async def _run(self) -> None:
        while True:
            try:
                rows = self._store.list_event_opinions_since(self._last_generated_at, limit=64)
                for op in rows:
                    await self._bus.publish(op.to_public_dict())
                    self._last_generated_at = op.generated_at
            except asyncio.CancelledError:
                raise
            except Exception:
                logger.exception("tailer_poll_failed")
            await asyncio.sleep(self._interval)
```

This requires `Store.list_event_opinions_since(since_ts, limit)` — if it does not already exist on the Store (check — prompt 01 likely added `get_event_opinion` but maybe not this paginated tail), add it: return published (non-abstained) opinions with `generated_at > since_ts`, ordered ascending, limit N. Index on `generated_at` should already exist for the feed queries.

## Observability invariants

- Every cycle emits exactly one `cycle_complete` log line with the full `CycleReport` payload. No per-event log lines at `INFO` — those are `DEBUG`.
- On any exception the scheduler logs `*_failed` at `ERROR` with a stacktrace and continues. The loop never crashes the process.
- `currents_status.json` is always up to date after the most recent cycle (successful or otherwise). Use this as the liveness signal in prompt 16's health probe — if the file is older than 2× `CYCLE_SECONDS`, the scheduler is unhealthy.
- `/metrics` is always-available and cheap (simple COUNT queries, no joins).

## Prohibitions

- Do NOT introduce Redis, Kafka, Celery, or any other broker. The pipeline is one process writing to SQLite; the API process tails it. If the project needs scale, that's a later prompt after measurement.
- Do NOT run the scheduler inside the API process. They have different failure modes — the API should be restartable on its own and the scheduler's crashes should not knock down serving.
- Do NOT carry budget balances across hour boundaries. The ceiling is a blast radius cap, not a rolling quota.
- Do NOT add a global retry-until-success around `generate_opinion`. It already does up-to-2 retries internally (prompt 05); the scheduler marks failures and moves on.
- Do NOT publish to the SSE bus from the scheduler process. Only the API process publishes to its own in-memory bus via the tailer; otherwise clients miss frames during restarts of either side.

Do not proceed to any other work. Do not edit anything outside SCOPE.
