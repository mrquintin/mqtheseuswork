### `_x_client.py`

Two methods, both async, both backoff-on-429:

- `fetch_user_tweets(user_id, since_id=None, max_results=20)` → list of normalized tweet dicts
- `search_recent(query, max_results=25)` → same shape

Use `httpx.AsyncClient` with `timeout=request_timeout_s`. On 429, read `x-rate-limit-reset`, sleep until it, retry once. On other 4xx, raise `XAPIError` with the body. On 5xx, retry up to 3 times with exponential backoff.

Normalized tweet shape:

```python
@dataclass(frozen=True)
class XPost:
    id: str
    text: str
    author_id: str
    author_handle: str
    created_at: str   # ISO-8601 UTC
    url: str
```

### `dedupe.py`

```python
def dedupe_hash(text: str, url: str | None) -> str:
    """sha256 over the normalized text+url. Normalization:
       - lower-case
       - collapse whitespace
       - strip URLs from the text body before hashing (keeps semantic id stable
         when the same article is shared with different shortlinks)
       - strip @mentions and hashtags
    """
```

Uniqueness across the URL'd version and the bare-text version means RTs and quote-tweets don't get re-opined.

### `x_ingestor.py`

```python
@dataclass
class IngestReport:
    cycle_id: str
    fetched: int
    new_event_ids: list[str]
    duplicates: int
    errors: list[str]

async def ingest_once(store: Store, cfg: IngestorConfig) -> IngestReport:
    client = make_client(cfg)        # test seam — prompt 22 mocks this
    new_ids: list[str] = []
    errors: list[str] = []
    fetched = 0
    duplicates = 0
    for user_id in cfg.curated_accounts:
        try:
            posts = await client.fetch_user_tweets(user_id)
            fetched += len(posts)
            for p in posts:
                if _persist_or_skip(store, cfg, p, "X_TWITTER", new_ids):
                    pass
                else:
                    duplicates += 1
        except Exception as e:
            errors.append(f"user:{user_id}:{type(e).__name__}: {e}")
    for q in cfg.search_queries:
        try:
            posts = await client.search_recent(q)
            fetched += len(posts)
            for p in posts:
                if _persist_or_skip(store, cfg, p, "X_TWITTER", new_ids):
                    pass
                else:
                    duplicates += 1
                if len(new_ids) >= cfg.max_events_per_cycle:
                    break
        except Exception as e:
            errors.append(f"query:{q!r}:{type(e).__name__}: {e}")
    return IngestReport(...)


def _persist_or_skip(store, cfg, post, source, out: list) -> bool:
    """Returns True if the post was persisted, False if skipped (dup)."""
    h = dedupe_hash(post.text, post.url)
    if store.find_current_event_by_dedupe(h):
        return False
    eid = store.add_current_event(CurrentEvent(
        organization_id=cfg.organization_id,
        source=source,
        external_id=post.id,
        author_handle=post.author_handle,
        text=post.text,
        url=post.url,
        observed_at=datetime.fromisoformat(post.created_at),
        dedupe_hash=h,
        status="OBSERVED",
    ))
    out.append(eid)
    return True
```

### `make_client` test seam

Module-level factory: `def make_client(cfg) -> XClient`. The pipeline e2e test (prompt 22) monkeypatches this to return a `FakeXClient`. Do NOT inline `XClient(...)` constructions inside `ingest_once`.

### Tests

`noosphere/tests/test_x_ingestor.py`:

- `dedupe_hash` is stable: same text+url → same hash; whitespace differences → same hash.
- `ingest_once` with a fake client returning 3 posts → 3 new events; second call → 0 new (all dedupe).
- An XAPIError on one curated account doesn't kill the whole cycle; the report's `errors` list captures it.
- `make_client` raises `MissingCredentials` when `X_BEARER_TOKEN` is empty.
