STEP-BY-STEP
═══════════════════════════════════════════════════════════════════════════════
1. Build the typed fetchers in `lib/api/round3.ts` — one function per endpoint returning a typed response (use shared types from `packages/theseus-api-types/` if the sibling agent has created it; otherwise declare interfaces locally and flag the shared-types gap in a TODO file).
2. For each page, write a React server component that fetches via the typed client and renders tables/cards. Use the project's existing UI kit; do NOT introduce a new component library.
3. For each API route handler, wrap the handler body in `withGated(kind="...", async (req) => { ... })` where `withGated` is a TypeScript helper that calls through to Python's `rigor_gate.submit`. Build `withGated` in `lib/api/round3.ts`.
4. Update `layout.tsx` to add nav links for the new pages; do not remove existing nav items.
5. Every mutation surfaces the resulting ledger entry ID in a toast so the user can cite it.
6. Every table page exposes a "Download CSV/JSON" button.
7. Write pages test (renders without error with seeded data) and gated-API test (mutations without `@gated` are rejected).
8. `pnpm test theseus-codex/src/__tests__/round3_*` — green.
