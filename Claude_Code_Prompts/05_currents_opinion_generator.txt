### 1. `_opinion_prompts.py` — the system and user templates

The system prompt is strict. Publish it in a single Python string so behavior is reproducible.

```python
OPINION_SYSTEM_PROMPT = """\
You are Theseus's current-events commentator. Theseus is an intellectual-capital
firm whose public knowledge base contains CONCLUSIONS (stable published beliefs)
and CLAIMS (raw founder, voice, and literature assertions). You do NOT hold
beliefs of your own; you only articulate what the firm's existing Noosphere
implies — or fails to imply — about a live current event.

Your reply is strict JSON matching this schema:

{
  "stance": "agrees" | "disagrees" | "complicates" | "insufficient",
  "confidence": number between 0.0 and 1.0,
  "headline": "one-sentence, neutral, 12-25 words",
  "body_markdown": "3-6 sentences of markdown, no more",
  "uncertainty_notes": ["…", "…"],
  "citations": [
    {
      "source_kind": "conclusion" | "claim",
      "source_id": "<exact id from SOURCES>",
      "quoted_span": "verbatim substring from that source, 8-240 chars",
      "relevance_score": number 0..1
    }
  ]
}

Hard rules:
- Every assertion in body_markdown must be traceable to at least one citation.
- Use ONLY the ids listed in SOURCES. Never invent an id.
- quoted_span MUST appear verbatim inside the corresponding SOURCE text.
- If the sources genuinely do not let you say anything substantive about the
  event, return stance="insufficient" with citations=[] and a body_markdown
  that says why the firm has no informed position.
- Do NOT editorialize beyond what the sources support.
- Do NOT include political slogans, moral pronouncements, or rhetorical
  flourishes unsourced.
- Do NOT use "we believe" or "we think". Use "the firm's prior conclusion
  suggests" or "existing Noosphere claims indicate".
- No emojis. No headers. No bullet lists inside body_markdown.
- Write in plain declarative English. Do not hedge excessively ("perhaps",
  "arguably", "it could be argued that") — the confidence field is the place
  to register uncertainty.

The stance label is coarse:
- "agrees": event aligns with what the firm already believes
- "disagrees": event contradicts something the firm already believes
- "complicates": event refines, qualifies, or reframes a firm belief
- "insufficient": sources do not support any of the above
"""

OPINION_USER_TEMPLATE = """\
EVENT
=====
Source: {source_url}
Author: {author_handle}
Captured at: {captured_at_iso}
Topic hint: {topic_hint}

RAW TEXT
--------
{raw_text}

SOURCES
=======
{sources_block}

Return the JSON object only. No prose before or after.
"""

def render_sources_block(hits) -> str:
    # Render each hit as:
    #   [SRC <n>] kind=<conclusion|claim> id=<source_id> score=<...>
    #   "<text>"
    lines = []
    for i, h in enumerate(hits, start=1):
        lines.append(f"[SRC {i}] kind={h.source_kind} id={h.source_id} score={h.score:.3f}")
        lines.append(f'"""')
        lines.append(h.text)
        lines.append(f'"""')
        lines.append("")
    return "\n".join(lines)
```

### 3. `budget.py` — hourly token ceiling

Public-facing opinion generation is continuous (every ~5 min). A budget guard prevents runaway cost if a burst of events coincides with a retrieval regression that floods the prompt. Hard-stop, not soft.

```python
@dataclass
class BudgetState:
    window_start: datetime
    prompt_tokens: int = 0
    completion_tokens: int = 0


class HourlyBudgetGuard:
    def __init__(self, max_prompt_tokens: int = 1_500_000, max_completion_tokens: int = 400_000):
        self.max_prompt = max_prompt_tokens
        self.max_completion = max_completion_tokens
        self._state = BudgetState(window_start=datetime.now(timezone.utc))

    def may_spend(self, est_prompt: int, est_completion: int) -> bool:
        self._roll_window()
        return (self._state.prompt_tokens + est_prompt <= self.max_prompt
                and self._state.completion_tokens + est_completion <= self.max_completion)

    def record(self, prompt: int, completion: int) -> None:
        self._roll_window()
        self._state.prompt_tokens += prompt
        self._state.completion_tokens += completion

    def _roll_window(self) -> None:
        now = datetime.now(timezone.utc)
        if now - self._state.window_start >= timedelta(hours=1):
            self._state = BudgetState(window_start=now)

    def snapshot(self) -> BudgetState:
        return BudgetState(
            window_start=self._state.window_start,
            prompt_tokens=self._state.prompt_tokens,
            completion_tokens=self._state.completion_tokens,
        )
```

Persist the budget state on disk (simple JSON at `noosphere_data/currents_budget.json`) so restarts don't reset the window. The scheduler (prompt 15) loads/saves.

### 4. `opinion_generator.py` — the orchestrator

```python
def generate_opinion(
    store: Store,
    event: CurrentEvent,
    *,
    budget: HourlyBudgetGuard,
    api_key: str | None = None,
    model: str = "claude-haiku-4-5",
    max_retries: int = 2,
) -> EventOpinion | None:
    """Returns the written EventOpinion, or None if we abstained or ran out of budget."""
    hits = retrieve_for_event(store, event)
    if not hits:
        store.update_current_event_status(event.id, CurrentEventStatus.ABSTAINED,
                                          reason="no_sources_at_generation_time")
        return None

    sanitized_text = PromptSeparator().clean(event.raw_text)   # strip injection
    user = OPINION_USER_TEMPLATE.format(
        source_url=event.source_url,
        author_handle=event.source_author_handle,
        captured_at_iso=event.source_captured_at.isoformat(),
        topic_hint=event.topic_hint or "(none)",
        raw_text=sanitized_text[:3000],
        sources_block=render_sources_block(hits),
    )

    est_prompt = estimate_tokens(OPINION_SYSTEM_PROMPT) + estimate_tokens(user)
    est_completion = 900
    if not budget.may_spend(est_prompt, est_completion):
        logger.warning("opinion_budget_exhausted", event_id=event.id)
        return None

    last_error = None
    for attempt in range(max_retries + 1):
        try:
            reply = chat_json(system=OPINION_SYSTEM_PROMPT, user=user, model=model,
                              max_tokens=est_completion, api_key=api_key)
        except LLMError as e:
            last_error = f"llm_error:{e}"
            continue

        budget.record(reply.tokens_prompt, reply.tokens_completion)

        ok, payload_or_err = _parse_and_validate(reply.text, hits, store)
        if ok:
            opinion, citations = _to_opinion_and_citations(
                event=event, payload=payload_or_err, hits=hits, reply=reply,
            )
            store.add_event_opinion(opinion, citations)
            store.update_current_event_status(event.id, CurrentEventStatus.OPINED, reason=None)
            logger.info("opinion_ok", event_id=event.id, opinion_id=opinion.id,
                        citations=len(citations))
            return opinion
        last_error = payload_or_err

    store.update_current_event_status(event.id, CurrentEventStatus.ABSTAINED,
                                      reason=f"opinion_validation_failed:{last_error[:120]}")
    return None
```

### 5. `_parse_and_validate` — this is the citation firewall

Must enforce all of:

1. Response is valid JSON with the declared schema.
2. `stance ∈ {agrees, disagrees, complicates, insufficient}`.
3. `confidence ∈ [0, 1]`.
4. `headline` is 30–180 chars, no newlines.
5. `body_markdown` is ≤ 800 chars.
6. For every citation:
   - `source_id` is in the hit set (`{h.source_id for h in hits}`).
   - `source_kind` matches that hit's `source_kind`.
   - `quoted_span` appears verbatim (case-insensitive, whitespace-normalized) inside the hit's full stored text (NOT just the truncated retrieval `text` — look up the underlying Conclusion/Claim and verify against the full body).
   - `relevance_score` ∈ [0, 1].
7. If `stance != "insufficient"`, `len(citations) >= 1`.
8. If `stance == "insufficient"`, `citations == []` and the event is not worth persisting an opinion for — return `False` with reason `insufficient_stance` so the caller abstains.

Any failure returns `(False, reason_string)`. On success returns `(True, parsed_dict)`.

### 6. `_to_opinion_and_citations`

Build `EventOpinion` via `make_opinion_id(event.id, model, generated_at.isoformat())` and `OpinionCitation` rows via `make_citation_id(opinion.id, ordinal)`. `sources_considered = len(hits)`, `sources_cited = len(citations)`.

## Prohibitions

- Do NOT skip the quoted-span substring check. A valid `source_id` with fabricated quoted text defeats the entire citation surface.
- Do NOT use a model other than `claude-haiku-4-5` by default. Don't silently upgrade to Sonnet/Opus if you think the output would be better — quality is policed by the validator, cost is policed by the budget.
- Do NOT let the LLM choose the `source_id` format. Ids come from the retrieval hits; the prompt makes this explicit; the validator enforces it.
- Do NOT write partial state on failure. Every write is transactional via `Store.add_event_opinion`.
- Do NOT log the full prompt or LLM reply at INFO level in production. Truncate to 200 chars. (Log at DEBUG if needed for dev.)
- Do NOT add a fallback path that lowers requirements on retry. Retries use the same prompt — the LLM is a nondeterministic renderer, not a requirement-relaxer.
