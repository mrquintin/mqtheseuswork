## Step-by-step

1. **Create `dialectic/dialectic/resources.py`** — a module that resolves asset paths correctly in both development and frozen (PyInstaller) modes:
   ```python
   import sys
   from pathlib import Path

   def is_frozen() -> bool:
       return getattr(sys, 'frozen', False) and hasattr(sys, '_MEIPASS')

   def base_path() -> Path:
       if is_frozen():
           return Path(sys._MEIPASS)
       return Path(__file__).resolve().parent.parent

   def asset_path(relative: str) -> Path:
       return base_path() / "assets" / relative

   def data_dir() -> Path:
       """User-writable data directory for sessions, logs, config."""
       if sys.platform == "darwin":
           d = Path.home() / "Library" / "Application Support" / "Dialectic"
       elif sys.platform == "win32":
           d = Path(os.environ.get("APPDATA", Path.home())) / "Dialectic"
       else:
           d = Path.home() / ".dialectic"
       d.mkdir(parents=True, exist_ok=True)
       return d
   ```
   Add the missing `import os` at the top. Ensure `data_dir()` handles all three platforms.

2. **Update `dialectic/dialectic/config.py`** — wherever file paths are resolved (model paths, device configs, session save directories), import from `resources.py` so they work in frozen mode. If `config.py` currently uses `Path(__file__)` or relative paths for assets, replace those with calls to `asset_path()` or `data_dir()`. Do NOT change any configuration values, thresholds, or device settings — only path resolution.

3. **Update `dialectic/dialectic/__main__.py`** and `dialectic/run.py`** — add a frozen-mode guard at the very top (before Qt or audio imports):

   This is required for PyInstaller on Windows where multiprocessing spawns subprocesses.
