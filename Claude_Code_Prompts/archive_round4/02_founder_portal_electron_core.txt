## Your task

You are an implementation agent. Your job is to add Electron desktop-app support to the Founder Portal. The Founder Portal is a Next.js 16 / React 19 web application with a SQLite database (via Prisma 7 + better-sqlite3). You will create an Electron main process that:
- Starts the Next.js standalone server as a child process
- Opens a BrowserWindow pointing to the local server
- Manages the SQLite database location in the platform-appropriate app-data directory
- Handles app lifecycle (graceful shutdown, single-instance lock, dock/taskbar behavior)

When you finish, verify the Electron main process compiles and the dev workflow launches. Do not proceed to any other work.

## Step-by-step

1. **Create `founder-portal/desktop/db-path.ts`** — resolves the SQLite database path per platform:
   ```typescript
   import { app } from 'electron';
   import path from 'path';
   import fs from 'fs';

   export function getDbPath(): string {
     const userDataDir = app.getPath('userData');
     const dbDir = path.join(userDataDir, 'data');
     fs.mkdirSync(dbDir, { recursive: true });
     return path.join(dbDir, 'founder-portal.db');
   }

   export function getDatabaseUrl(): string {
     return `file:${getDbPath()}`;
   }
   ```

2. **Create `founder-portal/desktop/next-server.ts`** — manages the Next.js standalone server lifecycle:
   ```typescript
   import { ChildProcess, fork } from 'child_process';
   import path from 'path';
   import { app } from 'electron';
   import net from 'net';

   let serverProcess: ChildProcess | null = null;

   async function findFreePort(startPort: number = 3000): Promise<number> { /* ... */ }

   export async function startNextServer(dbUrl: string): Promise<number> {
     const port = await findFreePort(3000);
     const standaloneDir = path.join(app.getAppPath(), '.next', 'standalone');
     const serverScript = path.join(standaloneDir, 'server.js');

     serverProcess = fork(serverScript, [], {
       env: {
         ...process.env,
         PORT: String(port),
         HOSTNAME: '127.0.0.1',
         DATABASE_URL: dbUrl,
         NODE_ENV: 'production',
       },
       stdio: 'pipe',
     });

     // Wait for server to be ready (poll the port)
     await waitForServer(port, 30_000);
     return port;
   }

   export function stopNextServer(): void { /* graceful kill */ }
   ```
   Implement `waitForServer` by polling `net.connect` to the port with exponential backoff (max 30s). Implement `stopNextServer` with SIGTERM, then SIGKILL after 5s timeout. Handle `serverProcess.on('exit')` to detect crashes.

3. **Create `founder-portal/desktop/preload.ts`** — minimal context bridge:
   ```typescript
   import { contextBridge, ipcRenderer } from 'electron';

   contextBridge.exposeInMainWorld('electronAPI', {
     platform: process.platform,
     versions: {
       electron: process.versions.electron,
       node: process.versions.node,
       chrome: process.versions.chrome,
     },
     onDeepLink: (callback: (url: string) => void) =>
       ipcRenderer.on('deep-link', (_event, url) => callback(url)),
   });
   ```

4. **Create `founder-portal/desktop/main.ts`** — the Electron main process:
   - Import `app`, `BrowserWindow`, `shell` from `electron`.
   - Enforce single instance with `app.requestSingleInstanceLock()`.
   - On `app.whenReady()`:
     a. Call `getDatabaseUrl()` to resolve the DB path.
     b. If this is the first launch (DB file doesn't exist), run Prisma migrations programmatically OR copy a seed database. For simplicity, shell out to `npx prisma migrate deploy` with the correct DATABASE_URL, or bundle a pre-migrated empty DB.
     c. Call `startNextServer(dbUrl)` to get the port.
     d. Create a `BrowserWindow` with:
        - `width: 1400, height: 900, minWidth: 1024, minHeight: 700`
        - `webPreferences: { preload: path.join(__dirname, 'preload.js'), contextIsolation: true, nodeIntegration: false }`
        - `titleBarStyle: 'hiddenInset'` on macOS for native look
     e. Load `http://127.0.0.1:${port}`.
     f. Open external links in the system browser via `shell.openExternal`.
   - On `app.on('window-all-closed')`: quit on all platforms (macOS too — this is a work app, not a general utility).
   - On `app.on('before-quit')`: call `stopNextServer()`.
   - On `app.on('second-instance')`: focus the existing window.
