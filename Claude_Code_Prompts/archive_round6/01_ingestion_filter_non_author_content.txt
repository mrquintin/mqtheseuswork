## The problem

When a founder uploads content to the Codex, the ingestion pipeline extracts "claims" — atomic truth-apt propositions — and stores them as conclusions. But the extraction is indiscriminate: it treats ALL text as the founder's own position. This means:

1. **Questions asked TO the founder** (e.g., interview prompts, debate challenges) get extracted as if they are the founder's conclusions. If a prompt says "Isn't your position wrong because X?", the system may extract "the position is wrong because X" as a founder conclusion.

2. **Debate positions contrary to the founder** get extracted as founder claims. If the text contains a counterargument the founder is responding to, that counterargument may be extracted alongside the founder's actual response, creating false contradictions.

3. **Quoted or paraphrased external positions** that the founder is analyzing or criticizing get treated as the founder's own views.

This is catastrophic for an epistemic system — it pollutes the founder's belief model with positions they may actively disagree with, and generates spurious contradictions between the founder's actual views and the external content misattributed to them.

## Your task

Modify the claim extraction pipeline so that it distinguishes between:
- **Founder assertions**: Claims the author is actually endorsing or asserting
- **External content**: Questions, prompts, debate positions, quoted opposing views, paraphrased challenges

External content should either be excluded from extraction entirely or extracted with `claim_origin=ClaimOrigin.EXTERNAL` (you'll need to add this enum value) so downstream systems can filter it out.
