Behavior when applied to `fn`:

  2. REGISTRY.register(spec, wrapped_fn).
  3. Insert into Store: `Store.insert_method(spec)` (idempotent on method_id).
  4. Return the wrapped_fn.

WRAPPED CALL FLOW (what `wrapped_fn(input)` does):
  1. Validate input: if input_schema is a pydantic model class, parse it; if it's a JSON schema dict, validate via jsonschema.
  2. Compute `input_hash = sha256(canonical_json(input))` hex digest.
  3. Construct `inv = MethodInvocation(id=uuid4(), method_id=spec.method_id, input_hash=input_hash, output_hash="", started_at=now, ended_at=None, succeeded=False, error_kind=None, correlation_id=_current_correlation_id(), tenant_id=_current_tenant_id())`.
  4. For each (name, hook) in _PRE_HOOKS, call hook(spec, inv, input). If any raises, jump to FAILURE PATH with that exception.
  5. Try: result = fn(input).
     - Validate result against output_schema.
     - output_hash = sha256(canonical_json(result)).
     - inv.ended_at = now; inv.succeeded = True; inv.output_hash = output_hash.
     - For each (name, hook) in _POST_HOOKS, call hook(spec, inv, input, result). Exceptions in post-hooks are logged but not re-raised.
     - Store.insert_method_invocation(inv).
     - Return result.
  6. FAILURE PATH: inv.ended_at = now; inv.succeeded = False; inv.error_kind = type(e).__name__.
     - For each (name, hook) in _FAILURE_HOOKS, call hook(spec, inv, input, e).
     - Store.insert_method_invocation(inv).
     - Re-raise the original exception.

Helpers:
  _current_git_sha() — read via `git rev-parse HEAD`; fall back to env var `THESEUS_GIT_SHA`; fall back to "unknown".
  _current_correlation_id() — read from a `contextvars.ContextVar` named `CORRELATION_ID` (create it if missing).

## Tests
In `test_method_registry.py`:
- Registering two versions of the same name both succeed; `get("x", "latest")` returns the active one with the highest semver.
- A retired method is not returned without `include_retired=True`.
- Registering the same (name, version) twice raises `MethodCollisionError`.
- `get("unknown")` raises `MethodNotFoundError`.
- Canonical input hashing: two calls with equal payloads produce equal input_hash.

In `test_method_hooks.py`:
- Pre-hooks fire in registration order.
- Post-hooks fire in registration order after the function body.
- A pre-hook raising aborts execution, runs failure hooks, still inserts a `succeeded=False` invocation, re-raises.
- A post-hook raising logs but does not re-raise (the call still returns).
- `unregister_hook` removes a hook cleanly.
- Re-registering a hook name replaces it in place.
