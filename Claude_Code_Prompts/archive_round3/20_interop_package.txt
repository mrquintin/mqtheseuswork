## Step-by-step
1. Write `docs/interop/MIP_SPEC.md` documenting the on-disk layout: manifest.json, methods/, cascade/, gate/, ledger/, docs/, LICENSE, CITATION.cff.
2. Build `builder.py` with `build_mip(methods, include_gate_checks, name, version, out_dir) -> MIPPath`. It invokes `transfer.package_method` for each method, copies their outputs into `methods/`, uses `docgen.compile_method_doc` to produce the docs, writes the manifest with content hashes, signs via `ledger.keys.KeyRing`.
3. Build `workflow.py` with a minimal YAML parser:
     - Allowed keys: `name`, `steps`, `output`.
     - Each step: `id`, `method`, `input`, `when` (equality predicate only).
     - No shell-out. No conditionals beyond equality. No loops.
     - `validate(workflow_yaml, available_methods) -> list[str]` returns validation errors.
4. Build `runner.py` with `run_mip(mip_path, workflow_name, input, out_dir)`:
     verifies manifest signature and checksums, parses workflow, runs steps via Docker (each method in its own container), writes local ledger entries, produces a run report.
5. Build `adoption.py` with `scaffold_adoption(mip_path, out_dir)`: writes a ready-to-run example directory, stub adapter, local scoreboard setup, README explaining how to cite the MIP.
6. Build `submit_transfer.py` accepting a `TransferStudy` produced by adopters; passes through `rigor_gate.submit` for review before display on the firm's public transfer-profile page.
7. Build `leak_check.py` scanning the bundle against a deny-list of firm-private identifiers + filenames; fails on hit.
8. Write tests.
9. `pytest noosphere/tests/test_mip_*.py -v`.

## Tests
test_mip_manifest.py — build a MIP with two methods; manifest valid; checksums verify; tampering a method file fails verification.
test_mip_runner.py — run a sample MIP on a clean host via Docker; produces a report with valid metrics; no network calls.
test_mip_workflow_validation.py — malformed workflows (missing step id, unknown method, disallowed key) produce specific errors; valid workflow validates clean.
test_mip_adoption_scaffold.py — `scaffold_adoption` produces a directory whose example run succeeds.
test_mip_license_coherence.py — conflicting sub-licenses inside the bundle fail build unless declared.
