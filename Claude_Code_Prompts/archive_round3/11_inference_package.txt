## Your task
You are an implementation agent. Your job is to create `noosphere/noosphere/inference/`: a package that, given a resolved real-world event, back-propagates through the corpus to identify supporting + refuting claims and produces a structured blindspot report. The inverse engine must use the `CorpusSlicer` from wave 4 for historical fidelity: a query with `as_of=T` sees only corpus items available as of T.

## Step-by-step
1. Build `inverse.py` with `InverseInferenceEngine.run(query: InverseQuery) -> InverseResult`, decorated as a registered method (name="inverse_inference", version="1.0.0", method_type=AGGREGATION, nondeterministic=True).
2. Construct a `CorpusSlicer(store, TemporalCut(as_of=query.as_of, ...))` and pass it to internal ranking so out-of-slice reads raise.
3. For each candidate claim in the slice, compute:
     entailment_score = nli_scorer(ClaimPair(claim, event_as_claim)).entail_prob
     refutation_score = nli_scorer(ClaimPair(claim, event_as_claim)).contradict_prob
     relevance_weight = softmax(cosine(embed(claim), embed(event)))
     impact = relevance_weight * max(entailment_score, refutation_score)
4. Return top-k by impact partitioned into `supporting` (entailment > refutation) and `refuted` (refutation > entailment), plus a blindspot report.
5. Build `blindspot.py` with `compute_blindspot(event, sliced_store) -> BlindspotReport`:
   - Missing entities: entities in event not present in any sliced claim.
   - Missing mechanisms: LLM-extracted mechanism tags in event not present in any sliced claim.
   - Adjacent-empty topics: topics embedding-adjacent to event with zero claims.
6. Emit `ResearchSuggestion` objects via the registered `suggest_research` method (routed through REGISTRY) so blindspot findings automatically propose reading lists.
7. Write the three test files.
8. `pytest noosphere/tests/test_inverse_*.py -v`.

## Tests
