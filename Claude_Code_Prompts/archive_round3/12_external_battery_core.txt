## Step-by-step
1. In `adapters/__init__.py`, define a `runtime_checkable` Protocol `CorpusAdapter`:
     class CorpusAdapter(Protocol):
         name: str
         license: LicenseTag
         def fetch(self, cache_dir: Path) -> CorpusBundle: ...
         def iter_items(self, bundle: CorpusBundle) -> Iterator[ExternalItem]: ...
         def resolve(self, item: ExternalItem, bundle: CorpusBundle) -> Outcome | None: ...
2. Build `canonical.py` with `canonicalize(item: ExternalItem) -> ClaimOrPrediction` converting external items into internal canonical form so registered methods operate uniformly.
3. Build `failures.py` with a five-way taxonomy: off_topic / mis_extraction / calibrated_but_wrong / confidently_wrong / hallucinated_dependency. Provide `classify_failure(item, method_output, resolution) -> FailureKind` and per-(method, corpus) histogram helpers.
4. Build `run.py` with `BatteryRunner.run(adapter, methods, sample_size) -> BatteryRunResult`, decorated as a registered method (name="external_battery_run", nondeterministic=True). For each item:
   a. Set `correlation_id` prefix to `cf-ext-<corpus>-...` so the evaluation embargo hook kicks in.
   b. Construct a `TemporalCut(as_of=item.as_of, ...)`.
   c. Call each method with the item canonicalized and `frozen_cut_id=cut.cut_id` in context.
   d. Compare method output to `adapter.resolve(item, bundle)`.
   e. Compute metrics via `evaluation.metrics`.
   f. Classify failures.
   g. Write report to `docs/eval/external/<corpus>/<date>/`.
   h. Emit red-flag ticket (via Store) if Brier regression past threshold vs prior run.
5. Write `docs/eval/external_corpora.md` as a stub listing the adapters that wave_6 will ship, with license notes per corpus.
6. Write `docs/eval/external/README.md` explaining the directory layout.
7. Write the three tests using a mock adapter you inline in `tests/conftest.py` or a helper module.
