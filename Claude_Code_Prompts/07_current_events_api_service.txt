## Your task

Create a new FastAPI app `current_events_api/` that exposes:

- `GET /health` — liveness
- `GET /v1/currents` — paginated list of `EventOpinion`s with filters
- `GET /v1/currents/{opinion_id}` — opinion detail with joined citations
- `GET /v1/currents/{opinion_id}/sources` — full Conclusion/Claim text for every citation
- `GET /v1/currents/stream` — SSE stream emitting newly-generated opinions (for the live ticker)
- `POST /v1/currents/{opinion_id}/follow-up` — start a session or continue one (SSE stream of the answer)
- `GET /v1/currents/{opinion_id}/follow-up/{session_id}/messages` — replay chat history (for permalinks and refresh)

Per-IP rate limiting (sliding window), structured JSON logging, graceful shutdown. The service does NOT own the noosphere Store directory — it reads the path from `NOOSPHERE_DATA_DIR` env var.

### 4. `event_bus.py` — in-process pub/sub

```python
import asyncio
from typing import AsyncIterator

class OpinionBus:
    def __init__(self):
        self._subs: list[asyncio.Queue[dict]] = []

    def publish(self, payload: dict) -> None:
        for q in list(self._subs):
            try:
                q.put_nowait(payload)
            except asyncio.QueueFull:
                pass

    async def subscribe(self) -> AsyncIterator[dict]:
        q: asyncio.Queue[dict] = asyncio.Queue(maxsize=64)
        self._subs.append(q)
        try:
            while True:
                yield await q.get()
        finally:
            self._subs.remove(q)
```

The scheduler (prompt 15) publishes to this bus when a new opinion is written. The bus is in-process; if you later run multiple replicas, swap in Redis pub/sub — but for now a single-process service matches the research-preview scale.

Stream shape (SSE events):

- `event: meta` — JSON `{"session_id": "...", "message_id": "..."}`
- `event: token` — plain text delta in `data:`

- `event: done` — JSON `{"refused": bool, "refusal_reason": "..."}`
- `event: error` — JSON `{"error": "..."}`

### 7. `routes/stream.py`

```python
@router.get("/currents/stream")
async def stream_currents(bus: OpinionBus = Depends(get_bus)):
    async def gen():
        async for payload in bus.subscribe():
            yield format_sse("opinion", payload)
    return StreamingResponse(gen(), media_type="text/event-stream")
```

`payload` is a `PublicOpinion.model_dump()`. The scheduler publishes full opinions, not just IDs — the UI renders without a second round trip.

Also emit `event: heartbeat` every 15s to keep proxies from buffering.

### 8. `rate_limit.py` — sliding-window per IP

Memory-backed (fine for single replica; swap to Redis later). Limits:

- `GET /v1/currents*`: 120/min per IP
- `POST /v1/currents/*/follow-up`: 10/min per IP (+ the engine's own daily/session caps applied inside `answer_followup`)
- `GET /v1/currents/stream`: 1 concurrent connection per IP

Return `429` with `Retry-After` on breach.

### 9. `sse.py` — tiny helper

```python
def format_sse(event: str, data: dict | str) -> str:
    body = data if isinstance(data, str) else json.dumps(data, default=str)
    return f"event: {event}\ndata: {body}\n\n"
```
