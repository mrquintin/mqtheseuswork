## The problem

The Overview tab for a conclusion shows three "MetaBlock" boxes:
- "Supporting principles: 3"
- "Evidence chain claims: 5"
- "Dissenting claims: 2"

These are just counts. The user cannot see WHAT the supporting principles say, WHAT the evidence chain claims are, or WHAT the dissenting claims argue. The IDs are parsed from JSON strings but never resolved to text. This is the most important tab for understanding a conclusion, and it shows the least useful information.

## Your task

Replace the count-only MetaBlocks with expandable sections that show the actual text of each principle, evidence claim, and dissenting claim. Keep the counts visible as summary labels, but add the ability to see the underlying content.

2. **Add resolver functions** in `round3.ts` (or in the page file if simpler). For each ID type, create a batch resolver that takes a list of IDs and returns `{ id, text }[]`. If the data is in the Codex DB:
   ```typescript
   export async function resolveConclusionTexts(ids: string[]): Promise<{ id: string; text: string }[]> {
     if (ids.length === 0) return [];
     const rows = await db.conclusion.findMany({
       where: { id: { in: ids } },
       select: { id: true, text: true },
     });
     return rows;
   }
   ```
   If the data is only in Noosphere, use a raw SQL query against the Noosphere tables, or provide a graceful fallback showing the IDs with a "Noosphere connection required for full text" note.

3. **In the conclusion detail page**, after fetching the conclusion, resolve all three ID arrays:
