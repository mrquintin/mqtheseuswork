## The problem

The contradictions page displays each contradiction as:
```
87% · cm3abc12… ↔ cm3def34…
```
These truncated CUIDs are meaningless to the founder. The page is unusable for understanding what actually contradicts what. The founder described the contradictions tab as "extremely messy" and said "it's not actually working properly" — this is the most visible symptom.

Each contradiction row has `claimAId` and `claimBId` that reference Conclusion records. The actual claim text is stored on those Conclusion rows. The page needs to resolve these IDs to their text and display the actual claims.
