## The problem

The contradictions page is a flat, unstyled list of `<details>` elements with minimal visual hierarchy. The founder called it "extremely messy." Even after prompts 06–07 fix the content (claim text + layer explanations), the page still needs structural and visual improvements:

1. No page header explanation of what contradictions are or why they matter.
2. No severity filtering or grouping — all 50 contradictions appear in a single flat list.
3. No visual distinction between high-severity (genuine concern) and low-severity (minor tension) contradictions.
4. The `<details>` expand/collapse pattern is functional but provides no visual affordance.
5. No connection back to the conclusions that are in tension — no links to navigate to the individual conclusion detail pages.

## Step-by-step

1. **Add a page introduction** below the title, matching the portal's Garamond + parchment style:
   ```tsx
   <p style={{
     fontFamily: "'EB Garamond', serif",
     fontStyle: "italic",

     maxWidth: "44em",
     lineHeight: 1.55,
     marginBottom: "1.5rem",
   }}>
     Contradictions surface when two conclusions in the model are in
     tension — they resist being held simultaneously. Each is scored by

     confident these claims genuinely conflict.
   </p>
   ```

2. **Group contradictions by severity band.** Define three bands:
   - **Critical** (severity ≥ 0.7): genuine contradictions requiring resolution
   - **Notable** (severity ≥ 0.4 and < 0.7): tensions worth reviewing
   - **Minor** (severity < 0.4): weak signals, possibly false positives

   Render each group under its own `ascii-frame` section header:
   ```tsx
   const critical = rows.filter((r) => r.severity >= 0.7);
   const notable = rows.filter((r) => r.severity >= 0.4 && r.severity < 0.7);
   const minor = rows.filter((r) => r.severity < 0.4);
   ```

   ```tsx
   {critical.length > 0 && (
     <section className="ascii-frame" data-label={`CRITICAL · ${critical.length}`}
       style={{ marginBottom: "1.5rem" }}>
       {/* render critical rows */}
     </section>
   )}
   {notable.length > 0 && (
     <section className="ascii-frame" data-label={`NOTABLE · ${notable.length}`}
       style={{ marginBottom: "1.5rem" }}>
       {/* render notable rows */}
     </section>
   )}
   {minor.length > 0 && (
     <section className="ascii-frame" data-label={`MINOR · ${minor.length}`}>
       {/* render minor rows */}
     </section>
   )}
   ```

   ```tsx

     padding: 0,
     overflow: "hidden",
   ```
