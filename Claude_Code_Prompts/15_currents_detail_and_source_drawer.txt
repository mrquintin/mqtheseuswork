# Claude Code Prompt 15 of 22 — Detail page `/currents/[id]` with source drawer + audit trail

**Sequence:** prompt 15 of 22 (Wave 5). Depends on 13, 14. Report back with: drawer layout, hash-link behavior (`#src-{id}`), how revoked sources are surfaced.

**Why this prompt exists.** Clicking an opinion card needs to land on a page that shows the full opinion with its sources fully expanded — verbatim quoted spans, links to the canonical Conclusion or Claim, and an audit trail (model, generated_at, retrieval scores, revocation status). This is what makes the "source-grounded" claim auditable.

**Before you start — read current state:**
1. Read `theseus-codex/src/lib/currentsApi.ts` — `getCurrent` and `getCurrentSources`.
2. Read `theseus-codex/src/app/c/[slug]/page.tsx` — the existing Conclusion detail page; the source drawer's "go to canonical" links target `/c/{slug}` URLs.

## Your task

### `/currents/[id]/page.tsx` (server)

```tsx
export const dynamic = "force-dynamic";

export default async function CurrentDetailPage({ params }) {
  const [op, sources] = await Promise.all([
    getCurrent(params.id),
    getCurrentSources(params.id),
  ]);
  return <DetailClient opinion={op} sources={sources} />;
}
```

### `DetailClient.tsx`

Two-column layout (desktop) / stacked (mobile):

- Left column: `<AuditTrail>` — opinion metadata (model, generated_at, retrieval scores per citation, revocation status).
- Center column: full opinion (headline + body markdown), then "Sources" heading, then full citations expanded as `<SourceCard>` instances.
- Right column (sticky): `<SourceDrawer>` that opens to a specific source when its citation chip is clicked or the URL has `#src-{id}`.

### `<SourceCard>`

Shows: source kind ("conclusion" / "claim"), full source text, verbatim quoted span highlighted (use the `highlight.ts` helper), retrieval score, "go to canonical" link to `/c/{slug}` (Conclusion) or `/conclusions/{id}#claim-{cid}` (Claim).

If `is_revoked`, show a strikethrough + amber "Revoked: {reason}" banner. Don't hide it — revocation context is part of the audit.
