# Claude Code Prompt 16 of 22 — Follow-up chat panel: streaming SSE, anonymous session, citation chips inline

**Sequence:** prompt 16 of 22 (Wave 5 closer). Depends on 15. Report back with: chat layout, the sessionStorage key format, citation rendering inline.

**Why this prompt exists.** Detail-page visitors should be able to ask follow-up questions about an opinion. The follow-up chat:

- Anonymous sessions stored in `sessionStorage` (24h TTL).
- POSTs `{question, session_id?}` to `/api/currents/[id]/follow-up`, gets back an SSE stream.
- Renders streaming tokens, then citation chips inline (linking to `/c/{slug}` for the conclusion or to the conclusion's claim anchor).
- Enforces a client-side 2-second min interval between sends to avoid hitting the server rate limiter accidentally.
- Shows "typing" indicator while awaiting first token.

2. Read `theseus-codex/src/app/currents/[id]/DetailClient.tsx` (prompt 15) — the chat mounts under the `#ask` anchor.

### `<FollowupChat>`

Client component, rendered at the bottom of `DetailClient`. State machine:

```
idle → awaiting_first_token → streaming → done
                ↓
               error
```

UI:
- Chat history (user/assistant alternating).
- User messages: plain text.
- Assistant messages: rendered via `safeMarkdown` from prompt 13.
- Inline citation chips: when a `citation` event arrives, append a chip showing `⸺ {source_kind}` linking to the canonical source page.
- Input: textarea + send button. Enter sends; Shift+Enter newline.
- Typing dots while awaiting first token.

Client-side guards:
- Min 2s between sends; the button disables for that interval.
- Question length capped at 1000 chars; live counter under the textarea.
- On 429 from server, surface the retry-after as a tooltip.

`#ask` URL anchor scrolls to the textarea and focuses it.
